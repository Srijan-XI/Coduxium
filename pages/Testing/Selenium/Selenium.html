Introduction

Selenium is a browser automation toolkit built around the WebDriver standard. Your test code controls a real browser (Chrome, Firefox, Edge), interacting with the page like a user: clicking buttons, typing into inputs, and asserting the UI state.

When to use Selenium

  • Validate critical user journeys (login, checkout, forms)
  • Cross-browser smoke tests
  • Regression tests for UI and integration flows
  • Debugging tricky client-side issues by reproducing steps

Best practice mindset

  • Prefer stable selectors (e.g., data-testid) over fragile CSS/XPath
  • Use explicit waits (WebDriverWait) instead of sleep
  • Keep tests independent and deterministic

Installation & Setup

This guide uses Python, which is common for Selenium automation. If you prefer Java, the workflow is similar (JUnit/TestNG + Selenium WebDriver).

1) Install Python

Install Python 3.10+ and ensure it’s available in your terminal.

2) Create a project and install Selenium

# Create a folder
mkdir selenium-demo
cd selenium-demo

# Create & activate a venv (Windows PowerShell)
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# Install Selenium
pip install selenium

3) Get a browser + driver

Modern Selenium can manage drivers automatically in many setups, but in locked-down environments you may need to install the driver manually.

  • Chrome/Edge: install the browser; driver may be auto-managed
  • Manual option: download matching driver and add it to PATH

4) Quick smoke test

from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Chrome()
try:
		driver.get("https://example.com")
		heading = driver.find_element(By.TAG_NAME, "h1").text
		assert "Example" in heading
finally:
		driver.quit()

User Guide

Project structure (simple)

selenium-demo/
	.venv/
	tests/
		test_login.py
	requirements.txt

Use stable selectors

In your app, add attributes like data-testid:

<button data-testid="login-submit">Sign in</button>

Wait for elements (avoid sleep)

from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

wait = WebDriverWait(driver, 10)
submit = wait.until(EC.element_to_be_clickable((By.CSS_SELECTOR, "[data-testid='login-submit']")))
submit.click()

Fill forms and assert results

driver.get("https://your-app.test/login")

driver.find_element(By.NAME, "email").send_keys("user@example.com")
driver.find_element(By.NAME, "password").send_keys("correct-horse-battery-staple")

driver.find_element(By.CSS_SELECTOR, "[data-testid='login-submit']").click()

wait.until(EC.url_contains("/dashboard"))
assert "Dashboard" in driver.title

Run in headless mode (CI-friendly)

from selenium.webdriver.chrome.options import Options

options = Options()
options.add_argument("--headless=new")
options.add_argument("--window-size=1280,800")

driver = webdriver.Chrome(options=options)

Troubleshooting tips

  • If elements aren’t found, wait for them (explicit waits) and double-check selectors.
  • If clicks fail, ensure the element is clickable and not covered by overlays.
  • If driver fails to start, check browser/driver version compatibility and PATH.
