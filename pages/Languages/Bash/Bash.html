📚 Introduction

What is Bash?

Bash (Bourne Again SHell) is a command processor and scripting language widely used on Linux, macOS, and Unix-like systems. It lets you run commands interactively or automate tasks with scripts. Bash serves as the default shell for many distributions and underpins countless deployment pipelines, build scripts, and system maintenance routines.

Key Features

📝 Text-Driven — Plain-text input and output
🔗 Composable — Pipes, redirection, chaining
🌍 Portable — Runs across POSIX environments
🔧 Extensible — Integrate with grep, awk, sed, jq
⚡ Lightweight — No heavy runtime needed
🤖 Automation — Perfect for DevOps tasks

Why Choose Bash?

  • Universal: Available on virtually all Unix-like systems
  • Quick Iteration: No compile step, instant feedback
  • System Integration: Direct access to OS commands and utilities
  • CI/CD Essential: Build and deployment scripts
  • Glue Language: Combine powerful tools efficiently
⚠️ Note: Bash is not ideal for large-scale application logic—prefer Python, Go, or Node.js for complex modules.

macOS Installation

macOS ships with Bash (older version) and Zsh as default shell. You can install a newer Bash using Homebrew.

# Install Homebrew if missing
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install latest Bash
brew install bash

# Add new shell to allowed list
echo "/opt/homebrew/bin/bash" | sudo tee -a /etc/shells

# Change default shell (optional)
chsh -s /opt/homebrew/bin/bash

# Verify
bash --version

Linux Installation / Update

Most Linux distributions include Bash by default.

# Debian/Ubuntu
sudo apt update
sudo apt install --reinstall bash

# Fedora
sudo dnf install bash

# Arch
sudo pacman -S bash

# Verify

📖 User Guide

Variables

# Variable assignment (no spaces around =)
NAME="Coduxium XI"
COUNT=42

# Using variables
echo "Hello $NAME"
echo "Count: ${COUNT}"

# Command substitution
TODAY=$(date +%Y-%m-%d)
FILES=$(ls -1 | wc -l)

# Environment variables
export PATH="$PATH:/usr/local/bin"
echo $HOME
echo $USER

Conditionals

# If statement
VALUE=42
if [ "$VALUE" -gt 10 ]; then
    echo "High"
elif [ "$VALUE" -eq 10 ]; then
    echo "Equal"
else
    echo "Low"
fi

# Test operators
if [ -f "file.txt" ]; then
    echo "File exists"
fi

if [ -d "directory" ]; then
    echo "Directory exists"
fi

# String comparison
if [ "$NAME" = "Alice" ]; then
    echo "Hello Alice"
fi

# Modern test syntax
if [[ $VALUE > 10 && $NAME == "Alice" ]]; then
    echo "Both conditions true"
fi

Loops

# For loop
for f in *.log; do
    echo "Processing $f"
    grep ERROR "$f" >> errors.txt
done

# For loop with range
for i in {1..5}; do
    echo "Number: $i"
done

# While loop
COUNT=0
while [ $COUNT -lt 5 ]; do
    echo $COUNT
    ((COUNT++))
done

# Read file line by line
while IFS= read -r line; do
    echo "Line: $line"
done < file.txt

Functions

# Function definition
greet() {
    local name=$1
    echo "Hi, $name"
}

# Call function
greet "Scripter"

# Function with return value
add() {
    local sum=$(( $1 + $2 ))
    echo $sum
}

result=$(add 5 3)
echo "Result: $result"

# Function with multiple parameters
process_file() {
    local input=$1
    local output=$2
    
    if [ ! -f "$input" ]; then
        echo "Error: Input file not found"
        return 1
    fi
    
    cat "$input" > "$output"
    echo "Processed: $input -> $output"
}

Arrays

# Array declaration
servers=(app01 app02 db01)

# Access elements
echo ${servers[0]}      # First element
echo ${servers[@]}      # All elements
echo ${#servers[@]}     # Array length

# Iterate over array
for s in "${servers[@]}"; do
    echo "Deploying to $s"
done

# Add to array
servers+=(web01)

# Associative array (Bash 4+)
declare -A config
config[host]="localhost"
config[port]="8080"

echo ${config[host]}

Input/Output & Redirection

# Output redirection
echo "Hello" > file.txt        # Overwrite
echo "World" >> file.txt       # Append

# Error redirection
command 2> errors.log          # Redirect stderr
command &> all.log             # Redirect both stdout and stderr

# Input redirection
wc -l < file.txt

# Pipes
cat file.txt | grep "pattern" | sort | uniq

# Here document
cat << EOF > config.txt
host: localhost
port: 8080
EOF

# Here string
grep "error" <<< "$LOG_OUTPUT"

String Manipulation

# String length
NAME="Alice"
echo ${#NAME}              # 5

# Substring
TEXT="Hello World"
echo ${TEXT:0:5}           # Hello
echo ${TEXT:6}             # World

# Replace
echo ${TEXT/World/Bash}    # Hello Bash
echo ${TEXT//o/0}          # Hell0 W0rld (replace all)

# Remove prefix/suffix
FILE="document.pdf"
echo ${FILE%.pdf}          # document
echo ${FILE#*.}            # pdf

# Uppercase/lowercase
echo ${NAME^^}             # ALICE
echo ${NAME,,}             # alice

Safer Scripts

#!/usr/bin/env bash

# Strict mode
set -Eeuo pipefail

# -e: Exit on error
# -E: Inherit ERR trap
# -u: Error on undefined variable
# -o pipefail: Pipe fails if any command fails

# Error handling
trap 'echo "Error on line $LINENO"' ERR

# Cleanup on exit
cleanup() {
    echo "Cleaning up..."
    rm -f /tmp/tempfile
}
trap cleanup EXIT

# Script logic here
echo "Script running..."

Best Practices

  • Always quote variables: "$VAR"
  • Use #!/usr/bin/env bash shebang
  • Enable strict mode with set -Eeuo pipefail
  • Prefer $(command) over backticks
  • Use shellcheck for linting
  • Document parameters and environment variables
  • Keep scripts modular with functions
