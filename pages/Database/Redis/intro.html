Overview

Redis stores data in memory for lightning-fast reads/writes and supports persistence via snapshots (RDB) and append-only files (AOF). It offers replication, partitioning, Lua scripting, streams, and modules.

Performance

Single-threaded event loop with I/O multiplexing enables over a million ops/sec on commodity hardware.

Persistence

RDB snapshots and AOF logs can be combined for durability. Tunable fsync policies balance performance vs safety.

High Availability

Redis Sentinel provides monitoring/failover; Redis Cluster offers sharding with automatic rebalancing.

Data Structures

  • Strings: binary-safe, counters, bitmaps.
  • Lists: queues, stacks, streams (LPUSH, BRPOP).
  • Sets: unique items, set ops (SADD, SUNION).
  • Hashes: field-value maps (HSET, HGETALL).
  • Sorted Sets: rankings with scores (ZADD, ZRANGE).
  • Streams: durable log with consumer groups (XADD, XREADGROUP).
  • HyperLogLog: cardinality estimation.
  • Bloom/CF (via modules): probabilistic filters.

Essential Commands

# Keys and strings
SET user:1 "Ada"
GET user:1
INCR visits

# Lists
LPUSH tasks "build" "test" "deploy"
BRPOP tasks 0

# Sets
SADD tags devops db cache
SUNION tags other

# Hashes
HSET user:1 name "Ada" role "admin"
HGETALL user:1

# Sorted sets
ZADD leaderboard 100 "Ada" 80 "Bob"
ZRANGE leaderboard 0 -1 WITHSCORES

Use Cases

  • Caching with TTLs and cache-aside patterns
  • Rate limiting using INCR and expirations
  • Real-time analytics with streams
  • Leaderboards via sorted sets
  • Session storage and pub/sub messaging
