What is DynamoDB?

DynamoDB is AWS’s fully managed NoSQL database that provides predictable performance and seamless scaling. It offers global replication, point-in-time restore, in-memory acceleration with DAX, and integrated security with AWS IAM.

Why DynamoDB?

  • Performance: Single-digit ms latency at any scale.
  • Operationally simple: No servers to manage. Automatic scaling and fault tolerance.
  • Cost efficient: Provisioned or on-demand capacity modes.
  • Global: Global Tables for multi-region, active-active replication.

Core Concepts

Table — collection of items
Item — set of attributes (row)
Attribute — key–value (column)
Partition key — hash key
Sort key — range key (optional)
GSI — Global Secondary Index
LSI — Local Secondary Index
TTL — Time-to-live auto-expiration
Streams — Change data capture
Transactions — ACID ops across items
Modeling note: Design for your access patterns, not for relational joins.

Data Modeling Basics

Single-table design

Group related entity types into one table using composite keys and type prefixes to answer queries with a small number of indexed lookups.

{
	"PK": "USER#42",
	"SK": "PROFILE#2025-01-01",
	"Type": "UserProfile",
	"Name": "Ada",
	"City": "London"
}
{
	"PK": "USER#42",
	"SK": "ORDER#A1001",
	"Type": "Order",
	"Total": 79.90
}

Code Examples

Python (boto3)

import boto3

ddb = boto3.resource("dynamodb")
table = ddb.Table("users")

# Put item
table.put_item(Item={"PK": "USER#42", "SK": "PROFILE#2025", "name": "Ada"})

# Get item
resp = table.get_item(Key={"PK": "USER#42", "SK": "PROFILE#2025"})
print(resp.get("Item"))

Node.js (AWS SDK v3)

import { DynamoDBClient } from "@aws-sdk/client-dynamodb";
import { DynamoDBDocumentClient, PutCommand, GetCommand } from "@aws-sdk/lib-dynamodb";

const client = new DynamoDBClient({});
const doc = DynamoDBDocumentClient.from(client);

await doc.send(new PutCommand({
	TableName: "users",
	Item: { PK: "USER#42", SK: "PROFILE#2025", name: "Ada" }
}));

const { Item } = await doc.send(new GetCommand({
	TableName: "users",
	Key: { PK: "USER#42", SK: "PROFILE#2025" }
}));
console.log(Item);

Ecosystem

  • DAX: in-memory cache for microsecond read latency
  • Global Tables: multi-region active-active
  • Streams + Lambda: event-driven patterns
  • PartiQL: SQL-compatible query language
  • CloudWatch: metrics and alarms
