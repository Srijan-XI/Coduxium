📚 Introduction

What is MongoDB?

MongoDB is a document-oriented NoSQL database that stores data in flexible, JSON-like BSON documents. Unlike traditional relational databases, MongoDB's schema-free design allows you to store documents with varying structures in the same collection, making it ideal for modern, agile development.

Key Features

📄 Document Model — Store rich, hierarchical data structures naturally
🔄 Flexible Schema — Evolve your data model without downtime
🚀 High Performance — Optimized for both reads and writes
🌐 Horizontal Scaling — Built-in sharding for distributed data
🔁 Replica Sets — Automatic failover and data redundancy
📊 Aggregation Framework — Powerful data processing pipeline
🔍 Rich Query Language — Expressive queries with indexing support
💾 GridFS — Store files larger than 16MB

Why Choose MongoDB?

  • Developer Productivity: Work with data structures that match your application objects, reducing impedance mismatch.
  • Flexible Schema: Add fields on the fly without ALTER TABLE statements or database migrations.
  • Scalability: Horizontal scaling through sharding distributes data across multiple servers seamlessly.
  • High Availability: Replica sets provide automatic failover and data redundancy for business continuity.
  • Rich Ecosystem: Native drivers for 10+ languages, MongoDB Atlas for cloud deployment, and extensive tooling.
  • Performance: Memory-mapped files, efficient indexing, and query optimization deliver fast response times.

Use Cases

📱 Mobile Apps — User profiles, preferences, offline sync
🛒 E-commerce — Product catalogs, shopping carts, inventory
📰 Content Management — Articles, comments, metadata
📊 Real-time Analytics — Event logging, metrics, time-series data
🎮 Gaming — Player profiles, game state, leaderboards
🌐 IoT — Sensor data, device telemetry, monitoring

⚙️ Installation & Setup

Get started with MongoDB by installing MongoDB Community Server and the MongoDB Shell (mongosh) on your platform.

Windows Installation

Option 1 — Official MSI Installer (Recommended)

1 Download MongoDB Community Server from mongodb.com/downloads
2 Run the MSI installer and choose "Complete" installation
3 Select "Install MongoDB as a Service" and "Run service as Network Service user"
4 Include MongoDB Compass (GUI tool) if desired

Option 2 — winget

# Install MongoDB Server
winget install --id MongoDB.Server -e

# Install MongoDB Shell
winget install --id MongoDB.Shell -e

# Verify installation
mongosh --version

macOS Installation

# Add MongoDB tap
brew tap mongodb/brew

# Install MongoDB Community Edition
brew install mongodb-community@7.0

# Start MongoDB as a service
brew services start mongodb-community@7.0

# Verify MongoDB is running
brew services list

# Install MongoDB Shell (if not included)
brew install mongosh

# Verify installation
mongosh --version

Linux Installation

Ubuntu/Debian

curl -fsSL https://pgp.mongodb.com/server-7.0.asc | \

# Update package database
sudo apt-get update

# Install MongoDB
sudo apt-get install -y mongodb-org

# Start MongoDB service
sudo systemctl enable mongod
sudo systemctl start mongod

# Verify status
sudo systemctl status mongod

📖 User Guide

Learn the essential operations and best practices for working with MongoDB effectively.

CRUD Operations

Create (Insert)

// Insert one document
db.users.insertOne({
  name: "Ada Lovelace",
  email: "ada@example.com",
  age: 30,
  interests: ["mathematics", "programming"]
})

// Insert multiple documents
db.users.insertMany([
  { name: "Grace Hopper", email: "grace@example.com", age: 85 },
  { name: "Alan Turing", email: "alan@example.com", age: 41 }
])

// Insert with custom _id
db.users.insertOne({
  _id: "custom-id-123",
  name: "Custom ID User",
  email: "custom@example.com"
})

Read (Query)

// Find all documents
db.users.find()

// Find with filter
db.users.find({ age: { $gte: 25 } })

// Find one document
db.users.findOne({ email: "ada@example.com" })

// Projection (select specific fields)
db.users.find(
  { age: { $gte: 25 } },
  { name: 1, email: 1, _id: 0 }
)

// Limit and sort
db.users.find().sort({ age: -1 }).limit(5)

// Count documents
db.users.countDocuments({ age: { $gte: 25 } })

// Advanced queries
db.users.find({
  $or: [
    { age: { $lt: 30 } },
    { interests: "mathematics" }
  ]
})

Update

// Update one document
db.users.updateOne(
  { email: "ada@example.com" },
  { $set: { age: 31, verified: true } }
)

// Update multiple documents
db.users.updateMany(
  { age: { $lt: 30 } },
  { $set: { category: "young" } }
)

// Replace entire document
db.users.replaceOne(
  { email: "ada@example.com" },
  { name: "Ada L.", email: "ada@example.com", age: 31 }
)

// Upsert (insert if not exists)
db.users.updateOne(
  { email: "new@example.com" },
  { $set: { name: "New User", age: 25 } },
  { upsert: true }
)

// Update operators
db.users.updateOne(
  { email: "ada@example.com" },
  {
    $inc: { age: 1 },                    // Increment
    $push: { interests: "analytics" },   // Add to array
    $set: { lastModified: new Date() }   // Set field
  }
)

Delete

// Delete one document
db.users.deleteOne({ email: "ada@example.com" })

// Delete multiple documents
db.users.deleteMany({ age: { $lt: 18 } })

// Delete all documents in collection
db.users.deleteMany({})

// Drop entire collection
db.users.drop()

// Drop database
db.dropDatabase()

Indexing for Performance

Indexes improve query performance by reducing the number of documents MongoDB must scan.

// Create single field index
db.users.createIndex({ email: 1 })

// Create unique index
db.users.createIndex({ email: 1 }, { unique: true })

// Create compound index
db.events.createIndex({ organization: 1, timestamp: -1 })

// Create text index for full-text search
db.articles.createIndex({ content: "text", title: "text" })

// Create sparse index (only index documents with field)
db.users.createIndex({ phone: 1 }, { sparse: true })

// Create TTL index (auto-delete after expiration)
db.sessions.createIndex(
  { createdAt: 1 },
  { expireAfterSeconds: 3600 }
)

// View all indexes
db.users.getIndexes()

// Drop index
db.users.dropIndex("email_1")

// Analyze query performance
db.users.find({ email: "ada@example.com" }).explain("executionStats")

Aggregation Framework

The aggregation pipeline processes documents through multiple stages for complex data transformations.

// Basic aggregation pipeline
db.orders.aggregate([
  // Stage 1: Filter documents
  { $match: { status: "completed" } },
  
  // Stage 2: Unwind array
  { $unwind: "$items" },
  
  // Stage 3: Group and calculate
  { $group: {
    _id: "$items.product",
    totalQuantity: { $sum: "$items.quantity" },
    totalRevenue: { $sum: { $multiply: ["$items.quantity", "$items.price"] } },
    orderCount: { $sum: 1 }
  }},
  
  // Stage 4: Sort by revenue
  { $sort: { totalRevenue: -1 } },
  
  // Stage 5: Limit results
  { $limit: 10 }
])

// Lookup (join) example
db.orders.aggregate([
  { $lookup: {
    from: "customers",
    localField: "customerId",
    foreignField: "_id",
    as: "customerDetails"
  }},
  { $unwind: "$customerDetails" },
  { $project: {
    orderId: 1,
    customerName: "$customerDetails.name",
    total: 1,
    date: 1
  }}
])

// Advanced aggregation with multiple operators
db.sales.aggregate([
  { $match: {
    date: { $gte: ISODate("2025-01-01"), $lt: ISODate("2025-12-31") }
  }},
  { $group: {
    _id: { $month: "$date" },
    revenue: { $sum: "$amount" },
    avgTransaction: { $avg: "$amount" },
    maxTransaction: { $max: "$amount" },
    count: { $sum: 1 }
  }},
  { $sort: { "_id": 1 } },
  { $project: {
    month: "$_id",
    revenue: 1,
    avgTransaction: { $round: ["$avgTransaction", 2] },
    maxTransaction: 1,
    count: 1,
    _id: 0
  }}
])

Data Modeling Patterns

Embedding vs Referencing

// EMBEDDING (denormalization) - Related data in same document
// Use when: data is accessed together, 1-to-few relationships
{
  _id: ObjectId("..."),
  username: "ada",
  email: "ada@example.com",
  addresses: [
    { type: "home", street: "123 Main St", city: "London" },
    { type: "work", street: "456 Office Rd", city: "London" }
  ]
}

// REFERENCING (normalization) - Store references to other documents
// Use when: data is large, 1-to-many or many-to-many relationships
// User document
{
  _id: ObjectId("user123"),
  username: "ada",
  email: "ada@example.com"
}

// Order documents
{
  _id: ObjectId("order456"),
  userId: ObjectId("user123"),  // Reference to user
  items: [...],
  total: 99.99,
  date: ISODate("2025-12-23")
}

Common Patterns

// Pattern 1: Polymorphic Pattern
// Different document types in same collection
{ type: "book", title: "...", author: "...", isbn: "..." }
{ type: "article", title: "...", publication: "...", url: "..." }

// Pattern 2: Bucket Pattern
// Group time-series data
{
  sensor_id: "A1",
  date: ISODate("2025-12-23"),
  measurements: [
    { time: ISODate("2025-12-23T10:00:00Z"), temp: 20.5 },
    { time: ISODate("2025-12-23T10:01:00Z"), temp: 20.6 }
  ]
}

// Pattern 3: Subset Pattern
// Store frequently accessed subset
{
  _id: ObjectId("movie123"),
  title: "The Matrix",
  year: 1999,
  recentReviews: [...],  // Most recent 10 reviews
  totalReviews: 15342    // Full reviews in separate collection
}

🚀 Advanced Features

Replica Sets

Replica sets provide redundancy and high availability through automatic failover.

// Initialize replica set (run on primary)
rs.initiate({
  _id: "myReplicaSet",
  members: [
    { _id: 0, host: "mongodb0.example.com:27017" },
    { _id: 1, host: "mongodb1.example.com:27017" },
    { _id: 2, host: "mongodb2.example.com:27017" }
  ]
})

// Check replica set status
rs.status()

// Add new member
rs.add("mongodb3.example.com:27017")

// Connection string with replica set
mongodb://mongodb0.example.com:27017,mongodb1.example.com:27017,mongodb2.example.com:27017/?replicaSet=myReplicaSet

Sharding

Sharding distributes data across multiple machines for horizontal scaling.

// Enable sharding for database
sh.enableSharding("mydb")

// Shard a collection
sh.shardCollection("mydb.users", { userId: "hashed" })

// Check sharding status
sh.status()

// View shard distribution
db.users.getShardDistribution()

Transactions

Multi-document ACID transactions for complex operations.

// Start a session
const session = db.getMongo().startSession()

// Start transaction
session.startTransaction()

try {
  const ordersCol = session.getDatabase("mydb").orders
  const inventoryCol = session.getDatabase("mydb").inventory
  
  // Insert order
  ordersCol.insertOne({
    orderId: "ORD123",
    items: [{ sku: "ABC", qty: 5 }],
    total: 99.99
  })
  
  // Update inventory
  inventoryCol.updateOne(
    { sku: "ABC" },
    { $inc: { quantity: -5 } }
  )
  
  // Commit transaction
  session.commitTransaction()
} catch (error) {
  // Abort on error
  session.abortTransaction()
  throw error
} finally {
  session.endSession()
}

Change Streams

Watch real-time changes to collections for event-driven architectures.

// Watch all changes
const changeStream = db.users.watch()

// Process changes
changeStream.on("change", (change) => {
  console.log(change)
})

// Watch specific operations
const pipeline = [
  { $match: { operationType: "insert" } }
]
const insertStream = db.users.watch(pipeline)
