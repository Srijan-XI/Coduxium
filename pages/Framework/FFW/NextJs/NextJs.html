Overview

Next.js brings server-first React with file-system routing, data fetching on the server, and automatic code splitting. The App Router (app/) uses React Server Components for great performance and DX.

# Create a new app
npx create-next-app@latest my-app
cd my-app
npm run dev

Quick Start

      
Use the official starter to scaffold an app with sensible defaults.

      
npx create-next-app@latest my-app
# or
pnpm create next-app my-app
# or
yarn create next-app my-app

      
cd my-app
npm run dev

    


    

      
TypeScript

      
Pick TypeScript during setup, or enable later by adding tsconfig.json and renaming files to .ts/.tsx. Next.js auto-installs TS deps on first run.

      
// tsconfig.json (minimal)
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "jsx": "preserve",
    "strict": true,
    "baseUrl": ".",
    "paths": { "@/*": ["./*"] }
  }
}

Environment

      
Use .env.local for secrets (not committed). Prefix with NEXT_PUBLIC_ for variables exposed to the client.

      
# .env.local
DATABASE_URL=postgres://user:pass@localhost:5432/app
NEXT_PUBLIC_API_BASE=https://api.example.com

    


    

      
Build & Start

      
Build for production and start the server. Configure port with PORT or next.config.js when needed.

      
npm run build
npm run start -- -p 3000

    


    

      
Verify

      
npm run dev
# Visit http://localhost:3000
