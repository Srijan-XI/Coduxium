📚 Introduction

What is Nuxt.js?

Nuxt.js is a powerful, open-source Vue.js framework that simplifies the development of universal or single-page Vue applications. Created by the Nuxt team, it provides server-side rendering (SSR), static site generation (SSG), and SPA capabilities out of the box with a modular architecture and excellent developer experience.

Key Features

🎯 Server-Side Rendering — SEO-friendly by default
📁 File-Based Routing — Automatic route generation
⚡ Fast Performance — Code splitting, lazy loading
🔧 TypeScript Support — First-class TS integration
🌐 Universal Rendering — SSR, SSG, SPA modes
📦 Module System — Rich ecosystem of modules
🔄 Auto Imports — Components, composables auto-imported
🎨 Vue 3 Powered — Latest Vue features

Why Choose Nuxt.js?

  • SEO Optimized: Built-in SSR for search engine visibility
  • Developer Experience: Hot reload, auto-imports, TypeScript
  • Flexible Rendering: Choose SSR, SSG, or SPA per route
  • Vue Ecosystem: Access to all Vue libraries and tools
  • Production Ready: Optimized builds, caching, preloading
  • Active Community: Large ecosystem of modules and plugins
⚠️ Important: Use Nuxt 3 for new projects. It's built on Vue 3, Vite, and offers better performance.

⚙️ Installation & Setup

Get started with Nuxt.js by creating a new project.

Create New Project

Using npx (Recommended)

# Create new Nuxt 3 project
npx nuxi@latest init my-nuxt-app

# Navigate to project
cd my-nuxt-app

# Install dependencies
npm install

# Start development server
npm run dev

Using pnpm

pnpm dlx nuxi@latest init my-nuxt-app
cd my-nuxt-app
pnpm install
pnpm dev

Using yarn

yarn dlx nuxi@latest init my-nuxt-app
cd my-nuxt-app
yarn install
yarn dev
💡 Note: The dev server will be available at http://localhost:3000

Project Structure

my-nuxt-app/
├── .nuxt/           # Build output (auto-generated)
├── assets/          # Uncompiled assets (CSS, images)
├── components/      # Vue components
├── composables/     # Vue composables
├── layouts/         # App layouts
├── middleware/      # Route middleware
├── pages/           # File-based routing
├── plugins/         # Plugins
├── public/          # Static files
├── server/          # Server API routes
├── app.vue          # Main app component
├── nuxt.config.ts   # Nuxt configuration
└── package.json     # Dependencies

TypeScript Setup

Nuxt 3 has built-in TypeScript support. Just use .ts or .vue with lang="ts":

// nuxt.config.ts is TypeScript by default
export default defineNuxtConfig({
  typescript: {
    strict: true,
    typeCheck: true
  }
})

Composables

// composables/useCounter.ts
export const useCounter = () => {
  const count = ref(0)
  const increment = () => count.value++
  const decrement = () => count.value--
  
  return {
    count: readonly(count),
    increment,
    decrement
  }
}

// Usage in component
<script setup>
const { count, increment } = useCounter()
</script>

<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="increment">+</button>
  </div>
</template>

Data Fetching

<script setup>
// useFetch - auto-imports
const { data: posts, pending, error } = await useFetch('/api/posts')

// useAsyncData with custom fetcher
const { data: user } = await useAsyncData('user', () => $fetch('/api/user/1'))

// Lazy loading
const { data: comments } = await useLazyAsyncData('comments', 
  () => $fetch('/api/comments')
)
</script>

<template>
  <div>
    <div v-if="pending">Loading...</div>
    <div v-else-if="error">Error: {{ error.message }}</div>
    <div v-else>
      <div v-for="post in posts" :key="post.id">
        {{ post.title }}
      </div>
    </div>
  </div>
</template>

    name: 'John Doe'
  }
})

// server/api/posts.post.ts - POST request

  // Save to database
  return { success: true, data: body }

Configuration

// nuxt.config.ts
export default defineNuxtConfig({
  modules: [
    '@nuxtjs/tailwindcss',
    '@nuxt/content',
    '@pinia/nuxt'
  ],
  
  app: {
    head: {
      title: 'My Nuxt App',
      meta: [
        { name: 'description', content: 'My amazing Nuxt app' }
      ]
    }
  },
  
  runtimeConfig: {
    // Private keys (server-side only)
    apiSecret: process.env.API_SECRET,
    
    // Public keys (exposed to client)
    public: {
      apiBase: process.env.API_BASE_URL || '/api'
    }
  }
})

State Management (Pinia)

// stores/counter.ts
export const useCounterStore = defineStore('counter', {
  state: () => ({
    count: 0
  }),
  
  actions: {
    increment() {
      this.count++
    }
  },
  
  getters: {
    doubleCount: (state) => state.count * 2
  }
})

// Usage in component
<script setup>
const counter = useCounterStore()
</script>

<template>
  <div>
    <p>{{ counter.count }}</p>
    <button @click="counter.increment">+</button>
  </div>
</template>
