🎈 Streamlit

The fastest way to build and share data apps - turn Python scripts into shareable web applications in minutes.

Overview

Data Apps Python Web Framework Rapid Development

Streamlit is an open-source Python library that makes it easy to create and share custom web apps for machine learning and data science. Build beautiful data apps in pure Python without frontend experience.

Key Features

  • Pure Python — No HTML, CSS, or JavaScript required
  • Instant Updates — See changes in real-time
  • Interactive Widgets — Sliders, buttons, text inputs, etc.
  • Data Visualization — Built-in support for charts
  • Caching — Speed up apps with @st.cache decorator
  • Deployment — Easy deployment to Streamlit Cloud
  • File Operations — Upload and download files easily
  • Customization — Themes and custom components

Common Use Cases

  • Machine learning model demos
  • Data exploration dashboards
  • Interactive reports
  • Internal tools and prototypes
  • Data science portfolio projects
  • Business intelligence applications
  • A/B testing tools
  • Data validation interfaces

Getting Started

Install Streamlit:

pip install streamlit

Create a simple app (app.py):

import streamlit as st
import pandas as pd
import numpy as np

st.title('My First Streamlit App')

st.write('Hello, Streamlit!')

# Create a dataframe
df = pd.DataFrame({
    'Column A': [1, 2, 3, 4],
    'Column B': [10, 20, 30, 40]
})

st.write('Here is a dataframe:')
st.dataframe(df)

# Add widgets
x = st.slider('Select a value', 0, 100)
st.write(f'You selected: {x}')

# Plotting
chart_data = pd.DataFrame(
    np.random.randn(20, 3),
    columns=['a', 'b', 'c']
)
st.line_chart(chart_data)

Run the app:

streamlit run app.py

Best Practices

  • Use @st.cache_data for data loading functions
  • Organize code with st.sidebar for controls
  • Use st.columns for layout
  • Implement session state for complex interactions
  • Add loading spinners for long operations
  • Use st.form to group inputs together
  • Deploy to Streamlit Cloud for easy sharing
  • Add error handling with try/except blocks
