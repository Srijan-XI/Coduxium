Axum

Axum is a modern Rust web framework built on Tokio and Tower, designed for ergonomic routing with strong typed extraction.

Overview

Backend Rust Tokio Tower

Axum is ideal for async Rust services requiring composable middleware, predictable architecture, and high confidence in type contracts.

Key Features

  • Typed extractors — safe request parsing primitives
  • Tower middleware — composable service layers
  • Tokio runtime — production-grade async foundation
  • Flexible routing — clean route composition patterns

Common Use Cases

  • Rust microservices and APIs
  • Concurrent backend workers with HTTP interfaces
  • Typed service boundaries for critical systems
  • Gateway and orchestration endpoints

Getting Started

cargo new axum-app
cd axum-app
cargo add axum tokio --features tokio/full
// src/main.rs
use axum::{routing::get, Router};

#[tokio::main]
async fn main() {
    let app = Router::new().route("/health", get(|| async { "ok" }));
    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

Best Practices

  • Use state extractors for explicit dependency boundaries.
  • Layer middleware with clear ordering and responsibilities.
  • Validate and normalize request payloads early.
  • Instrument handlers with tracing spans for observability.
