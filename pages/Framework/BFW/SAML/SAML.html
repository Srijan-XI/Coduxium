SAML

Security Assertion Markup Language (SAML) is an XML-based standard for exchanging authentication and authorization data between parties.

Overview

Integration Standard Enterprise SSO Identity Federation XML

SAML is widely used in enterprise identity environments where centralized identity providers authenticate users for multiple applications.

Core Concepts

  • Identity Provider (IdP) — authenticates users
  • Service Provider (SP) — consumes SAML assertions
  • Assertion — signed identity and attribute payload
  • Metadata — trusted configuration exchange

Common Use Cases

  • Enterprise application SSO
  • Federation with corporate identity systems
  • B2B portal authentication integration
  • Legacy identity ecosystems

Best Practices

  • Validate signatures, issuer, and audience on every assertion.
  • Use strict clock skew and assertion lifetime checks.
  • Rotate and monitor IdP/SP certificates.
  • Prefer signed and encrypted assertions for sensitive contexts.
