Webhooks

Webhooks deliver event data via HTTP callbacks, enabling near real-time integration between systems.

Overview

Integration Pattern HTTP Callbacks Event Driven Automation

Webhooks are widely used to notify external systems when business events occur, such as payment status updates or deployment events.

Core Concepts

  • Endpoint — receiver URL for event callbacks
  • Signature — HMAC verification for authenticity
  • Retries — delivery backoff on transient failures
  • Idempotency — safe duplicate handling

Common Use Cases

  • Payment events (created, succeeded, failed)
  • CI/CD or deployment notifications
  • CRM and ticketing synchronization
  • Audit and compliance event ingestion

Receiver Checklist

  • Verify request signatures before processing payloads.
  • Store event IDs to prevent duplicate side effects.
  • Respond quickly and process asynchronously when heavy work is needed.
  • Log delivery attempts with correlation IDs.
