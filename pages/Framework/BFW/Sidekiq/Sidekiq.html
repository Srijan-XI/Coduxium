Sidekiq

Sidekiq is a Redis-backed job processor for Ruby used to run asynchronous and scheduled background tasks.

Overview

Background Jobs Redis Ruby Queue Processing

It provides reliable offloading of long-running or retryable work from request-response paths, improving latency and resilience.

Core Concepts

  • Workers — classes that process jobs
  • Queues — priority channels for jobs
  • Retries — built-in failure backoff
  • Scheduling — delayed and periodic execution

Common Use Cases

  • Email delivery and notifications
  • Data sync and ETL tasks
  • Webhook processing and retries
  • Batch reporting jobs

Worker Example

class WelcomeEmailWorker
  include Sidekiq::Worker

  def perform(user_id)
    UserMailer.welcome(user_id).deliver_now
  end
end

Best Practices

  • Ensure jobs are idempotent and safe for retries.
  • Keep job payloads small and pass identifiers, not full objects.
  • Partition queues by priority and workload class.
  • Track queue latency and dead jobs with alerting.

Related Backend Frameworks

  • Sails.js (comparable queue patterns)
  • NestJS (job queue alternatives)
  • Phoenix (Oban ecosystem equivalent)
