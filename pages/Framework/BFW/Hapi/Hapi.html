hapi

hapi is a configuration-centric Node.js framework focused on robust APIs, plugin architecture, and secure defaults for enterprise services.

Overview

Backend Config Driven Enterprise Plugins

hapi favors explicit configuration for routes, validation, and lifecycle extensions, making service behavior clear and maintainable.

Key Features

  • Route options — rich per-route config model
  • Built-in validation patterns — integrates well with Joi
  • Plugin architecture — compose reusable capabilities
  • Lifecycle extensions — fine-grained request hooks

Common Use Cases

  • Enterprise API platforms
  • Security-sensitive backend services
  • Config-heavy multi-team backends
  • Long-lived internal service ecosystems

Getting Started

mkdir hapi-app
cd hapi-app
npm init -y
npm i @hapi/hapi
node server.js
// server.js
const Hapi = require('@hapi/hapi');
const init = async () => {
  const server = Hapi.server({ port: 3000 });
  server.route({ method: 'GET', path: '/health', handler: () => ({ ok: true }) });
  await server.start();
};
init();

Best Practices

  • Keep route options explicit and documented.
  • Use Joi schemas for payload and params validation.
  • Register plugins by domain boundaries.
  • Use lifecycle hooks for observability and policy enforcement.
