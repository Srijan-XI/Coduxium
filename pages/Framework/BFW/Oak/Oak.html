Oak

Oak is a middleware framework for Deno inspired by Koa, built around the Fetch API and modern web standards.

Overview

Backend Deno Middleware Web Standards

Oak is ideal for Deno-first backend services where teams want explicit middleware flow, typed handlers, and simple routing APIs.

Key Features

  • Middleware pipeline — async composition for request lifecycle
  • Router support — route definitions with params and methods
  • Deno native tooling — no external package manager required
  • TypeScript by default — strong typing from day one

Common Use Cases

  • Deno-native REST APIs
  • Internal backend tools and services
  • Learning Deno middleware architecture
  • Lightweight service gateways

Getting Started

deno init oak-app
cd oak-app
deno add jsr:@oak/oak
// main.ts
import { Application, Router } from '@oak/oak'
const app = new Application()
const router = new Router()
router.get('/health', (ctx) => {
  ctx.response.body = { ok: true }
})
app.use(router.routes())
app.use(router.allowedMethods())
await app.listen({ port: 8000 })

Best Practices

  • Keep middleware small and focused on one responsibility.
  • Validate request payloads at boundary handlers.
  • Use explicit permission flags in production scripts.
  • Pin dependencies and runtime versions for reproducibility.
