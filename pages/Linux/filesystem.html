Filesystem Hierarchy Standard (FHS)

In Linux, everything is a file. The directory structure follows a standard layout known as the FHS. Understanding this is crucial for navigation.

The Root Directory (/)

/ The start of the filesystem tree
├── bin Essential command binaries (ls, cp, cat)
├── boot Bootloader files (kernel, grub)
├── dev Device nodes (sda, tty, null)
├── etc System configuration files (apt, systemd, ssh)
├── home User personal directories (Documents, Downloads)
├── lib Shared libraries for binaries
├── mnt Temporary mount point
├── opt Optional software packages
├── proc Virtual filesystem for process & kernel info
├── root Home directory for the root superuser
├── sbin System binaries (reboot, ip, fdisk)
├── tmp Temporary files (deleted on reboot)
├── usr User utilities and applications (/usr/bin)
└── var Variable files (logs, web root, cache)

File Permissions

Linux controls access via permissions for Owner, Group, and Others.

r (4) Read
w (2) Write
x (1) Execute

Calculations

Permissions are often represented as 3 numbers (e.g., 755).

  • 7 = 4+2+1 (Read/Write/Execute)
  • 5 = 4+0+1 (Read/Execute)
  • 6 = 4+2+0 (Read/Write)
# Change owner (user:group)
sudo chown user:group filename

# Make a file executable
chmod +x script.sh

# Set specific permissions (rwx for owner, rx for others)
chmod 755 filename

Common Filesystems

  • ext4: The standard, stable default for most Linux distributions.
  • Btrfs: Modern copy-on-write filesystem with snapshot support (default in Fedora, OpenSUSE).
  • XFS: High-performance journaling filesystem (default in RHEL).
  • FAT32/exFAT: Used for USB drives for compatibility with Windows/macOS.
