Accounts & Permissions

Use sudo, avoid shared accounts, and keep file permissions tight.

Core checks:
  • Disable unused accounts
  • Use strong passwords or SSH keys
  • Review sudoers entries
📦

Updates & Repos

Patch the OS and apps. Only use trusted repositories and signed packages.

Best practice:
  • Enable automatic security updates where appropriate
  • Pin/limit third‑party repos
🧱

Network Exposure

Open only required ports, limit listening services, and use a firewall.

Quick win:
  • Stop/disable unused services
  • Restrict SSH access

🛠️ Essential Security Commands

Updates

# Debian/Ubuntu
sudo apt update && sudo apt upgrade

# Fedora
sudo dnf upgrade

# Arch
sudo pacman -Syu

Firewall (UFW / Firewalld)

# UFW (Ubuntu)
sudo ufw enable
sudo ufw default deny incoming
sudo ufw allow OpenSSH

# Firewalld (Fedora/RHEL)
sudo firewall-cmd --state
sudo firewall-cmd --set-default-zone=public
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload

Listening Ports & Services

ss -tulpn
systemctl --type=service --state=running
journalctl -p warning..alert -b

🧰 SSH Hardening (Servers)

Recommended changes

Prefer SSH keys, disable root login, and reduce brute‑force exposure.

# Edit SSH configuration
sudo nano /etc/ssh/sshd_config

# Common hardening options
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes

# Reload service
sudo systemctl reload ssh
# or
sudo systemctl reload sshd

⚠️ Don’t lock yourself out

Keep an active session open while testing, and verify key login works before disabling passwords.

🛡️ SELinux & AppArmor

Feature SELinux AppArmor
Common on RHEL/Fedora/CentOS Ubuntu/Debian-based
Model Label-based policies Path-based profiles
Operating modes Enforcing / Permissive / Disabled Enforce / Complain / Disable

SELinux quick checks

sestatus
getenforce
sudo setenforce 0   # permissive (temporary)
sudo setenforce 1   # enforcing

AppArmor quick checks

sudo aa-status
sudo aa-complain /path/to/profile
sudo aa-enforce /path/to/profile

Logs

journalctl -xe
# SELinux often logs in audit
sudo ausearch -m avc -ts recent

🧯 Backups & Recovery

3-2-1 rule

Keep 3 copies, on 2 media types, with 1 offsite.

Test restores

A backup isn’t useful until a restore is proven.

Encrypt sensitive backups

Protect data at rest and in transit.

Document recovery

Write down steps, keys, and where backups live.
