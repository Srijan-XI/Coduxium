Linux Kernel Overview

The kernel is the interface between hardware and processes. It schedules CPU time, manages memory, handles filesystems and networking, and enforces permissions.

🧩 Core Responsibilities

🧠 Memory Management

Keeps track of memory usage. Handles virtual memory mapping and swapping.

⚡ Process Management

Schedules CPU time and manages process creation and termination.

🔌 Device Drivers

Bridges hardware and software. A large part of kernel code is drivers.

🛡️ System Calls & Security

Exposes services to processes (like opening files) and checks permissions.

🏗️ Monolithic vs Microkernel

Linux is a monolithic kernel: most OS services run in kernel space. Microkernels move more services into user space.

Tradeoff Monolithic (Linux) Microkernel
Performance High performance, fewer context switches Often more overhead, more IPC
Fault isolation Driver bugs can affect the system Better isolation for services

🧱 Kernel Modules

Linux is monolithic but modular: you can load and unload kernel modules without rebooting.

Module commands

# List loaded modules
lsmod

# Load a module
sudo modprobe name_of_module

# Remove a module
sudo modprobe -r name_of_module
