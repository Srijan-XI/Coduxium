🔧 Network Configuration

IP Configuration Commands

View Network Interfaces

ip addr show
# or
ifconfig

List all network interfaces

Set Static IP

sudo ip addr add 192.168.1.100/24 dev eth0
sudo ip link set eth0 up

Assign IP address to interface

Set Default Gateway

sudo ip route add default via 192.168.1.1

Configure default route

View Routing Table

ip route show
# or
route -n

Display routing information

Enable/Disable Interface

sudo ip link set eth0 up
sudo ip link set eth0 down

Control interface state

DHCP Configuration

sudo dhclient eth0
# or
sudo dhcpcd eth0

Obtain IP via DHCP

NetworkManager (nmcli)

Ubuntu Fedora RHEL

List Connections

nmcli connection show

Connect to Wi-Fi

nmcli device wifi connect SSID password PASSWORD

Create Connection

nmcli con add type ethernet con-name eth0-static ifname eth0 ip4 192.168.1.100/24 gw4 192.168.1.1

Modify Connection

nmcli con mod eth0-static ipv4.dns "8.8.8.8 8.8.4.4"

🔥 Firewall Configuration

UFW - Uncomplicated Firewall

Ubuntu Debian

Enable UFW

sudo ufw enable
sudo ufw status

Allow Port

sudo ufw allow 22/tcp
sudo ufw allow ssh

Allow from IP

sudo ufw allow from 192.168.1.100

Deny Port

sudo ufw deny 80/tcp

Delete Rule

sudo ufw delete allow 22

Reset Firewall

sudo ufw reset

firewalld

Fedora RHEL CentOS

Start firewalld

sudo systemctl start firewalld
sudo systemctl enable firewalld

List Rules

sudo firewall-cmd --list-all

Add Service

sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload

Add Port

sudo firewall-cmd --permanent --add-port=8080/tcp

Remove Service

sudo firewall-cmd --permanent --remove-service=http

List Zones

sudo firewall-cmd --get-zones
sudo firewall-cmd --get-active-zones

🔐 SSH Configuration

Connect to Server

ssh user@hostname
ssh user@192.168.1.100
ssh -p 2222 user@host

Generate SSH Key

ssh-keygen -t ed25519 -C "your@email.com"
# or
ssh-keygen -t rsa -b 4096

Copy Key to Server

ssh-copy-id user@host

SSH Config File

# ~/.ssh/config
Host myserver
    HostName 192.168.1.100
    User admin
    Port 22
    IdentityFile ~/.ssh/id_ed25519

SCP File Transfer

scp file.txt user@host:/path/
scp user@host:/path/file.txt ./

SSH Tunneling

ssh -L 8080:localhost:80 user@host
ssh -D 1080 user@host  # SOCKS proxy
🔒 SSH Security Best Practices 
# /etc/ssh/sshd_config
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
Port 2222  # Change default port
AllowUsers youruser

🌍 DNS Configuration

Set DNS Servers

# /etc/resolv.conf
nameserver 8.8.8.8
nameserver 8.8.4.4

DNS Lookup

nslookup google.com
dig google.com
host google.com

Test DNS Resolution

dig @8.8.8.8 google.com

Flush DNS Cache

sudo systemd-resolve --flush-caches
# or
sudo resolvectl flush-caches

🔍 Network Troubleshooting

Ping Test

ping google.com
ping -c 4 8.8.8.8

Test connectivity

Traceroute

traceroute google.com
# or
tracepath google.com

Trace network path

Port Scanning

nmap 192.168.1.1
nmap -p 80,443 192.168.1.1

Scan open ports

Network Statistics

netstat -tuln
ss -tuln

Show listening ports

Bandwidth Monitoring

iftop
nethogs
vnstat

Monitor traffic

Packet Capture

tcpdump -i eth0
tcpdump -w capture.pcap

Capture network packets

Check Connectivity

telnet google.com 80
nc -zv google.com 443

Test port connectivity

ARP Table

arp -a
ip neigh show

View ARP cache

MTU Test

ping -M do -s 1472 google.com

Test maximum packet size

✨ Best Practices

1

Use Static IPs for Servers

Configure static IP addresses for servers to ensure consistent accessibility.

2

Enable Firewalls

Always run a firewall and only open necessary ports.

3

Use SSH Keys

Disable password authentication and use SSH keys for secure access.

4

Monitor Network Traffic

Regularly monitor network traffic for unusual activity.

5

Document Network Config

Keep documentation of your network configurations and changes.

6

Regular Security Updates

Keep network tools and SSH updated with latest security patches.
