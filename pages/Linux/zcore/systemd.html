📚 Introduction

💡 What is Systemd?

Systemd is a modern init system and service manager for Linux that has become the de facto standard on most major distributions. It manages system boot, services, and provides powerful logging capabilities.

🔧 Service Management

Basic Service Commands

Start Service

sudo systemctl start nginx

Start a service immediately

Stop Service

sudo systemctl stop nginx

Stop a running service

Restart Service

sudo systemctl restart nginx

Restart a service

Reload Config

sudo systemctl reload nginx

Reload configuration without restart

Enable at Boot

sudo systemctl enable nginx

Start service on boot

Disable at Boot

sudo systemctl disable nginx

Prevent service from starting on boot

Check Status

systemctl status nginx

View service status and logs

Is Service Active?

systemctl is-active nginx

Check if service is running/enabled

List All Services

systemctl list-units --type=service

Show all services

📝 Creating Systemd Services

✨ Basic Service Unit File 
# /etc/systemd/system/myapp.service
[Unit]
Description=My Application
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/myapp
ExecStart=/usr/bin/python3 /opt/myapp/app.py
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target

Service Types

Type=simple    # Default
Type=forking   # Daemonizes
Type=oneshot   # Runs once
Type=notify    # Sends notification

Restart Policies

Restart=no
Restart=on-failure
Restart=always
Restart=on-abnormal

Environment Variables

[Service]
Environment="PORT=8080"
Environment="ENV=production"
EnvironmentFile=/etc/myapp/config

Dependencies

[Unit]
Requires=postgresql.service
After=postgresql.service
Before=nginx.service

Reload and Start Service

Reload Systemd

sudo systemctl daemon-reload

Reload systemd configuration

Enable & Start

sudo systemctl enable --now myapp

Enable and start in one command

Edit Service

sudo systemctl edit myapp

Edit service with drop-in file

⏰ Systemd Timers

💡 Timers vs Cron

Systemd timers are more powerful than cron - they can run immediately if a scheduled time was missed, are logged centrally, and integrate better with systemd services.

✨ Creating a Timer 
# /etc/systemd/system/backup.timer
[Unit]
Description=Daily Backup Timer

[Timer]
OnCalendar=daily
OnCalendar=*-*-* 02:00:00
Persistent=true

[Install]
WantedBy=timers.target
# /etc/systemd/system/backup.service
[Unit]
Description=Backup Service

[Service]
Type=oneshot
ExecStart=/usr/local/bin/backup.sh

Timer Schedules

OnCalendar=daily
OnCalendar=weekly
OnCalendar=monthly
OnCalendar=*-*-* 04:00:00

Relative Timers

OnBootSec=15min
OnUnitActiveSec=1h
OnActiveSec=30min

Enable Timer

sudo systemctl enable backup.timer
sudo systemctl start backup.timer

List Timers

systemctl list-timers --all

📊 Journalctl - System Logs

View All Logs

journalctl

Show all journal entries

Follow Logs

journalctl -f

Real-time log monitoring

Service Logs

journalctl -u nginx
journalctl -u nginx -f

Logs for specific service

Time Range

journalctl --since "1 hour ago"
journalctl --since "2025-01-01"
journalctl --since today

Priority Levels

journalctl -p err
journalctl -p warning

Filter by severity

Boot Logs

journalctl -b
journalctl -b -1  # Previous boot

Disk Usage

journalctl --disk-usage

Check log space usage

Clean Old Logs

sudo journalctl --vacuum-time=7d
sudo journalctl --vacuum-size=500M

Kernel Messages

journalctl -k

Kernel logs only

✨ Best Practices

1

Use Descriptive Names

Give your service files clear, descriptive names that explain their purpose.

2

Set Restart Policies

Always configure appropriate restart policies for critical services.

3

Use Timers Over Cron

Prefer systemd timers for scheduled tasks - better logging and reliability.

4

Monitor Logs Regularly

Use journalctl to monitor service logs and catch issues early.

5

Document Dependencies

Clearly define service dependencies with After/Before/Requires.

6

Clean Logs Periodically

Set up automatic journal cleanup to prevent disk space issues.

💼 Real-World Example

Web Application Service

# /etc/systemd/system/webapp.service
[Unit]
Description=My Web Application
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=notify
User=webapp
Group=webapp
WorkingDirectory=/var/www/myapp

# Environment
Environment="PORT=8000"
Environment="DATABASE_URL=postgresql://localhost/myapp"
EnvironmentFile=-/etc/webapp/config

# Commands
ExecStartPre=/usr/bin/python3 /var/www/myapp/migrate.py
ExecStart=/usr/bin/python3 /var/www/myapp/main.py
ExecReload=/bin/kill -HUP $MAINPID

# Restart policy
Restart=on-failure
RestartSec=5s
StartLimitBurst=3
StartLimitInterval=60s

# Security
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=full

# Logging
StandardOutput=journal
StandardError=journal
SyslogIdentifier=webapp

[Install]
WantedBy=multi-user.target
