Qubes OS

A security-focused operating system that isolates applications and workflows into separate virtual machines ("qubes") so a compromise in one area is less likely to spread to others.

📚 Introduction

What is Qubes OS?

Qubes OS is a security-oriented operating system that uses virtualization to implement security by compartmentalization. It runs applications in isolated compartments called qubes, so if one qube is compromised, the others are designed to remain isolated.

Key Features

  • App isolation using Xen virtual machines (qubes)
  • Color-coded trust levels for safer daily workflows
  • Disposable VMs for untrusted content
  • Hardware isolation for USB and networking components
  • Template-based maintenance for simpler patching

Why Choose Qubes OS?

1

Reduce Blast Radius

Compartmentalize work, personal, and untrusted activities to limit damage from compromise.

2

Flexible Templates

Use Fedora/Debian templates so you can choose tooling without losing consistency.

3

Privacy Workflows

Pair with Whonix and disposable VMs for privacy-first browsing and research.

⚙️ Installation Guide

Step 1: Download

Download Qubes OS from: https://www.qubes-os.org/downloads/

Step 2: Create Bootable USB

# Linux
sudo dd if=Qubes-R4.x-x86_64.iso of=/dev/sdX bs=1M status=progress && sync

# Or use: Etcher, Rufus (DD mode), Ventoy

Step 3: Boot from USB

1. Insert USB drive
2. Restart computer
3. Enter BIOS/UEFI (F2/F12/Del)
4. Select USB as boot device

Step 4: Installation Process

Test media (recommended), run the installer, enable full-disk encryption, complete partitioning, create user, finish installation, then reboot and complete initial qube setup.

⚠️ System Requirements
  • CPU: 64-bit with VT-x/AMD-V virtualization support
  • RAM: 6GB minimum (8GB+ recommended)
  • Storage: 32GB minimum (SSD recommended)
  • GPU: Any (depends on setup; integrated is fine)

📖 User Guide

Package Management

Update System

# dom0 updates
sudo qubes-dom0-update

# Fedora template
sudo dnf upgrade --refresh

# Debian template
sudo apt update && sudo apt upgrade

Install Package

# Run in a template VM
sudo dnf install package-name
# or
sudo apt install package-name

Remove Package

# Run in a template VM
sudo dnf remove package-name
# or
sudo apt remove package-name

Search Package

🚀 Advanced Topics

Whonix Integration

# Install Whonix templates (dom0)
sudo qubes-dom0-update qubes-template-whonix-gw
sudo qubes-dom0-update qubes-template-whonix-ws

# Create gateway/workstation qubes
qvm-create sys-whonix --template whonix-gw --label orange
qvm-create anon-whonix --template whonix-ws --label orange
qvm-prefs anon-whonix netvm sys-whonix

Route selected qubes through Tor via Whonix.

USB Device Isolation

# List USB devices
qvm-usb

# Attach a device to a qube
qvm-usb attach my-qube sys-usb:device-id

Keep USB attacks away from sensitive components.

Backup & Restore

# Back up via GUI (recommended):
# Qube Manager → System → Backup Qubes

# Restore (example)
qvm-backup-restore /path/to/backup

Use encrypted backups and store them safely offline.

Split GPG

# High-level workflow:
# 1) Keep private keys in a dedicated "vault" qube
# 2) Enable Split GPG service
# 3) Allow signing/encryption from other qubes without exposing keys

Protect private keys even if a work qube is compromised.

💻 Useful Commands

Task Command Description
List qubes qvm-ls Show all qubes and their states
Start qube qvm-start my-qube Boot a qube
Run app qvm-run my-qube firefox Launch an application inside a qube
Shutdown qube qvm-shutdown my-qube Stop a running qube
Copy file to qube qvm-copy-to-vm dest-qube /path/to/file Safely transfer files between qubes
Create qube qvm-create --class AppVM --label blue my-qube Create a new AppVM
List templates qvm-ls --class TemplateVM Show installed template VMs
Show qube prefs qvm-prefs my-qube Inspect configuration of a qube
Firewall settings qvm-firewall my-qube Configure per-qube firewall rules
Attach USB device qvm-usb attach my-qube dom0:BUS-DEV Attach a USB device to a qube
