📚 Introduction

What is Gentoo?

Gentoo is a source-based Linux distribution where most software is built locally from source code. This approach allows you to precisely tune features, dependencies, and compiler optimizations. Gentoo is especially popular with users who want maximum control or want to learn deeply how Linux systems fit together.

Core Concepts

  • Portage: Package manager (the emerge tool)
  • USE flags: Feature toggles that influence build options
  • Profiles: Sensible defaults for system targets
  • MAKEOPTS: Parallel compile settings (e.g., -j)
  • march / mtune: CPU-specific optimizations

⚙️ Installation Guide

Step 1: Pick a Stage & Handbook

Use the official Handbook and choose your architecture (amd64 is most common).

Step 2: Partition & Mount

# Example (adjust for your disk layout)
    lsblk
    sudo fdisk /dev/sdX

    sudo mkfs.ext4 /dev/sdX?
    sudo mount /dev/sdX? /mnt/gentoo

Step 3: Install Base System

Download/extract the stage tarball, configure chroot, then build kernel and install a bootloader.

Step 4: First Reboot

Configure networking, users, and desktop (optional), then reboot into your new Gentoo system.

⚠️ Heads-up
  • Expect longer install time due to compilation.
  • Use a fast network connection and adequate CPU/RAM.
  • Start with a minimal build; add desktop later.

📖 User Guide

Portage Basics

Sync Repos

sudo emerge --sync

Update System (World)

sudo emerge --ask --verbose --update --deep --newuse @world

Install Package

sudo emerge --ask app-editors/vim

Search Package

emerge --search nginx

USE Flags

USE flags enable/disable features across packages (e.g., enabling pulseaudio or wayland). You can set them globally in /etc/portage/make.conf or per-package in /etc/portage/package.use.

Inspect USE Flags

equery uses app-editors/vim
    emerge --pretend --verbose app-editors/vim

Per-package USE Example

# /etc/portage/package.use
    www-servers/nginx http2
    media-video/ffmpeg x264 x265

🚀 Advanced Topics

Compilation Tuning

# /etc/portage/make.conf
    MAKEOPTS="-j4"
    COMMON_FLAGS="-O2 -pipe -march=native"

Tune parallel builds and CPU optimizations carefully.

Kernel Configuration

cd /usr/src/linux
    make menuconfig
    make -j4 && make modules_install
    make install

Build a custom kernel for your system.

Cleaning & Maintenance

sudo emerge --depclean
    sudo revdep-rebuild

Remove unused deps and rebuild broken linkages.

Binary Packages (Optional)

# Some packages may be available as binaries
    emerge -g --ask package-name

Use binaries where available to save time.

💻 Useful Commands

Task Command Description
Sync Portage sudo emerge --sync Update the ebuild repository
Update world sudo emerge -avuDN @world Full system update with USE changes
Install package sudo emerge -a category/pkg Install from source via Portage
Search packages emerge --search keyword Find packages by name/description
Depclean sudo emerge --depclean Remove orphan dependencies
Pretend update sudo emerge -pvuDN @world Preview updates before compiling
Show system info emerge --info Display build configuration and environment
List profiles eselect profile list Show available Gentoo profiles
Set profile sudo eselect profile set <N> Select a system profile (careful!)
Inspect USE flags equery uses category/pkg See which USE flags a package supports
