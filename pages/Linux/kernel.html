What is the Linux Kernel?

The kernel is the core interface between a computer's hardware and its processes. It communicates between the 2, managing resources as efficiently as possible.

Core Responsibilities

🧠 Memory Management

Keeps track of how much memory is used and by what. Handles virtual memory mapping and swapping.

⚡ Process Management

Determines which process can use the CPU, when, and for how long. Handles process creation and termination.

🔌 Device Drivers

Acts as an interpreter between the hardware and software. Vast majority of kernel code is drivers.

🛡️ System Calls & Security

Receives requests from processes for service (like opening a file) and checks permissions.

Monolithic vs Microkernel

Linux is a Monolithic Kernel. This means the entire operating system runs in kernel space. This differs from microkernels (like Minix) where services run in user space.

  • Pros: High performance, direct access to hardware.
  • Cons: A crash in a driver can crash the whole system (though modern Linux is very robust).

Kernel Modules

Though monolithic, Linux is modular. You can load and unload code (modules) on the fly without rebooting.

# List loaded modules
lsmod

# Load a module
sudo modprobe name_of_module

# Remove a module
sudo modprobe -r name_of_module
