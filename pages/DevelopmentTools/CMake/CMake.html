📚 Introduction

CMake is a meta build system that generates native build files (Ninja, Makefiles, Visual Studio projects). It is the standard way to configure portable C/C++ projects and is widely used in embedded, desktop, and backend systems.

Core Benefits

  • Cross-Platform - same build definition across OS/toolchains
  • Target-Based Design - explicit include/link/interface dependencies
  • Toolchain Flexibility - GCC, Clang, MSVC, cross-compilers
  • Ecosystem Support - integrates with IDEs, package managers, CI

⚙️ Installation

# Windows (winget)
winget install Kitware.CMake

# macOS
brew install cmake

# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y cmake

📖 User Guide

Minimal CMakeLists.txt

cmake_minimum_required(VERSION 3.20)
project(hello LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_executable(hello src/main.cpp)

Build and Run

# Configure
cmake -S . -B build

# Build
cmake --build build

# Run (depends on generator/platform)
./build/hello

Modern Target Practice

add_library(core src/core.cpp)
target_include_directories(core PUBLIC include)

target_compile_definitions(core PRIVATE CORE_IMPL=1)

target_link_libraries(hello PRIVATE core)

🧠 Advanced Topics

Build Types and Generators

# Debug
cmake -S . -B build-debug -DCMAKE_BUILD_TYPE=Debug

# Release with Ninja
cmake -S . -B build-release -G Ninja -DCMAKE_BUILD_TYPE=Release

Best Practices

  • Prefer target-level commands over global flags
  • Keep third-party dependencies isolated and versioned
  • Use presets for team-wide consistent builds
  • Separate source and build directories
