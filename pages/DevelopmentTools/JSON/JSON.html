📄 Overview

JSON (JavaScript Object Notation) is a lightweight data-interchange format. It's completely language-independent but uses conventions familiar to programmers of C-family languages. It's the de facto standard for REST APIs and web data exchange.

Common Use Cases

  • REST APIs — Request/response payloads
  • Configuration Files — package.json, tsconfig.json
  • Data Storage — NoSQL databases (MongoDB)
  • Data Exchange — Between client and server

🔧 Working with JSON

JavaScript

// Parse JSON string
const jsonString = '{"name":"John","age":30}';
const obj = JSON.parse(jsonString);
console.log(obj.name); // "John"

// Convert object to JSON
const person = { name: "John", age: 30 };
const json = JSON.stringify(person);
console.log(json); // '{"name":"John","age":30}'

// Pretty print
const pretty = JSON.stringify(person, null, 2);

// With replacer
const filtered = JSON.stringify(person, ['name']);

Python

import json

# Parse JSON
json_string = '{"name": "John", "age": 30}'
data = json.loads(json_string)
print(data['name'])  # John

# Convert to JSON
person = {'name': 'John', 'age': 30}
json_string = json.dumps(person)

# Pretty print
pretty = json.dumps(person, indent=2)

# From/to file
with open('data.json', 'r') as f:
    data = json.load(f)

with open('output.json', 'w') as f:
    json.dump(data, f, indent=2)

Java

// Using Jackson
import com.fasterxml.jackson.databind.ObjectMapper;

ObjectMapper mapper = new ObjectMapper();

// Parse JSON
String json = "{\"name\":\"John\",\"age\":30}";
Person person = mapper.readValue(json, Person.class);

// Convert to JSON
String output = mapper.writeValueAsString(person);

// Pretty print
String pretty = mapper.writerWithDefaultPrettyPrinter()
                     .writeValueAsString(person);

C#

using System.Text.Json;

// Parse JSON
string json = "{\"name\":\"John\",\"age\":30}";
Person person = JsonSerializer.Deserialize<Person>(json);

// Convert to JSON
string output = JsonSerializer.Serialize(person);

// Pretty print
var options = new JsonSerializerOptions { WriteIndented = true };
string pretty = JsonSerializer.Serialize(person, options);

REST API Example

// Request
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
.then(res => res.json())
.then(data => console.log(data));

// Response
{
  "id": 123,
  "name": "John Doe",
  "email": "john@example.com",
  "created_at": "2025-01-01T00:00:00Z"
}

Best Practices

  • ✅ Use double quotes for keys and strings
  • ✅ No trailing commas
  • ✅ Use arrays for ordered data
  • ✅ Use objects for key-value pairs
  • ✅ Validate JSON with linters
  • ❌ Avoid comments (use separate docs)
  • ❌ Don't use functions or undefined

JSON vs Other Formats

  • vs XML: More concise, easier to read
  • vs YAML: Stricter syntax, better for APIs
  • vs TOML: Better for data exchange, less for config
