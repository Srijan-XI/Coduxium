⚡ Overview

Vite (French word for "quick", pronounced /vit/) is a build tool that significantly improves the frontend development experience. It serves code via native ES modules in development and bundles with Rollup for production, offering instant server start and lightning-fast Hot Module Replacement (HMR).

Key Features

  • Instant Server Start — No bundling in development
  • Lightning Fast HMR — Updates instantly regardless of app size
  • Optimized Build — Pre-configured Rollup build with best practices
  • Framework Support — React, Vue, Svelte, Vanilla JS, and more
  • TypeScript Support — Built-in TypeScript compilation
  • Plugin Ecosystem — Rich plugin system compatible with Rollup plugins

📦 Installation & Setup

Create New Project

# Create project with template
npm create vite@latest my-project

# Choose framework: vanilla, vue, react, preact, lit, svelte
# Choose variant: JavaScript or TypeScript

cd my-project
npm install
npm run dev

Framework-Specific Templates

# React + TypeScript
npm create vite@latest my-react-app -- --template react-ts

# Vue + TypeScript
npm create vite@latest my-vue-app -- --template vue-ts

# Svelte + TypeScript
npm create vite@latest my-svelte-app -- --template svelte-ts

# Vanilla TypeScript
npm create vite@latest my-vanilla-app -- --template vanilla-ts

Add to Existing Project

# Install Vite
npm install -D vite

# Add scripts to package.json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}

⚙️ Configuration

Basic vite.config.js

import { defineConfig } from 'vite'

export default defineConfig({
  // Server options
  server: {
    port: 3000,
    open: true,
    cors: true
  },
  
  // Build options
  build: {
    outDir: 'dist',
    sourcemap: true,
    minify: 'terser'
  },
  
  // Path aliases
  resolve: {
    alias: {
      '@': '/src',
      '@components': '/src/components'
    }
  }
})

React Configuration

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000
  },
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom']
        }
      }
    }
  }
})

Vue Configuration

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': '/src'
    }
  }
})

Environment Variables

# .env file
VITE_API_URL=https://api.example.com
VITE_APP_TITLE=My App

# Access in code
console.log(import.meta.env.VITE_API_URL)
console.log(import.meta.env.MODE) // 'development' or 'production'

# TypeScript support - vite-env.d.ts
interface ImportMetaEnv {
  readonly VITE_API_URL: string
  readonly VITE_APP_TITLE: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}

Using Plugins

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { VitePWA } from 'vite-plugin-pwa'
import viteCompression from 'vite-plugin-compression'

export default defineConfig({
  plugins: [
    react(),
    VitePWA({
      registerType: 'autoUpdate',
      manifest: {
        name: 'My App',
        short_name: 'App'
      }
    }),
    viteCompression({
      algorithm: 'gzip',
      ext: '.gz'
    })
  ]
})

Production Build

# Build for production
npm run build

# Preview production build locally
npm run preview

# Build with custom config
vite build --config vite.prod.config.js

# Analyze bundle size
npm install -D rollup-plugin-visualizer

import { visualizer } from 'rollup-plugin-visualizer'

export default defineConfig({
  plugins: [
    visualizer({
      open: true,
      gzipSize: true

  ]
})

Proxy Configuration

export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      },
      '/ws': {
        target: 'ws://localhost:8080',
        ws: true
      }
    }
  }
})

Library Mode

// Build as library
import { defineConfig } from 'vite'
import { resolve } from 'path'

export default defineConfig({
  build: {
    lib: {
      entry: resolve(__dirname, 'src/main.ts'),
      name: 'MyLib',
      fileName: (format) => `my-lib.${format}.js`,
      formats: ['es', 'cjs', 'umd']
    },
    rollupOptions: {
      external: ['react', 'react-dom'],
      output: {
        globals: {
          react: 'React',
          'react-dom': 'ReactDOM'
        }
      }
    }
  }
})

📊 Vite vs Other Build Tools

Vite vs Webpack

  • Dev Server: Vite is significantly faster (uses ESM)
  • HMR: Vite provides instant updates
  • Configuration: Vite requires minimal setup
  • Plugin Ecosystem: Webpack has more plugins, but Vite is growing fast

Vite vs Create React App

  • Speed: Vite is much faster in development
  • Flexibility: Vite is more configurable
  • Modern: Vite uses ESM, CRA uses traditional bundling
