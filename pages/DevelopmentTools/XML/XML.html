📄 Overview

XML (Extensible Markup Language) is a markup language that defines rules for encoding documents in a format that is both human-readable and machine-readable. It's widely used in enterprise applications, configuration files, and data exchange.

Common Use Cases

  • Configuration Files — Maven pom.xml, Spring beans
  • SOAP Web Services — Message format
  • Office Documents — .docx, .xlsx formats
  • RSS/Atom Feeds — Content syndication
  • Android Layouts — UI definitions

📖 XML Syntax

Basic Structure

<?xml version="1.0" encoding="UTF-8"?>
<root>
    <element>Value</element>
    <empty />
</root>

Elements and Attributes

<?xml version="1.0"?>
<person id="123" active="true">
    <name>John Doe</name>
    <age>30</age>
    <email>john@example.com</email>
</person>

Nested Elements

<?xml version="1.0"?>
<company>
    <name>Tech Corp</name>
    <employees>
        <employee id="1">
            <name>John</name>
            <title>Developer</title>
        </employee>
        <employee id="2">
            <name>Jane</name>
            <title>Manager</title>
        </employee>
    </employees>
</company>

CDATA Sections

<script>
    <![CDATA[
        function greeting() {
            if (x < 10) {
                return "Hello";
            }
        }
    ]]>
</script>

Comments

<!-- This is a comment -->
<element>
    <!-- Multi-line
         comment -->
    <value>text</value>
</element>

Namespaces

<?xml version="1.0"?>
<root xmlns:h="http://www.w3.org/TR/html4/"
      xmlns:f="https://www.example.com/furniture">
    <h:table>
        <h:tr>
            <h:td>Cell</h:td>
        </h:tr>
    </h:table>
    <f:table>
        <f:name>Dining Table</f:name>
    </f:table>
</root>

💼 Real-World Examples

Maven pom.xml

<project xmlns="http://maven.apache.org/POM/4.0.0">
    <modelVersion>4.0.0</modelVersion>
    
    <groupId>com.example</groupId>
    <artifactId>myapp</artifactId>
    <version>1.0.0</version>
    
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
            <version>3.2.0</version>
        </dependency>
    </dependencies>
</project>

Spring Configuration

    <bean id="dataSource"

</beans>

Parsing XML

// JavaScript
const parser = new DOMParser();
const xmlDoc = parser.parseFromString(xmlString, "text/xml");
const name = xmlDoc.getElementsByTagName("name")[0].textContent;

// Python
import xml.etree.ElementTree as ET

tree = ET.parse('file.xml')
root = tree.getroot()

for child in root:
    print(child.tag, child.attrib, child.text)

# Java
import javax.xml.parsers.DocumentBuilder;
import javax.xml.parsers.DocumentBuilderFactory;
import org.w3c.dom.Document;

DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
DocumentBuilder builder = factory.newDocumentBuilder();
Document doc = builder.parse(new File("file.xml"));

# C#
using System.Xml;

XmlDocument doc = new XmlDocument();
doc.Load("file.xml");
XmlNode node = doc.SelectSingleNode("/root/element");

Best Practices

  • ✅ Use meaningful element names
  • ✅ Proper indentation for readability
  • ✅ Close all tags properly
  • ✅ Use attributes for metadata
  • ✅ Validate with XSD schemas
  • ❌ Avoid deeply nested structures
  • ❌ Don't use special characters without escaping
