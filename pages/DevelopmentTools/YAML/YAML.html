📄 Overview

YAML is a human-readable data serialization language commonly used for configuration files and in applications where data is being stored or transmitted. It's widely used in DevOps, CI/CD pipelines, and configuration management.

Common Use Cases

  • CI/CD Pipelines — GitHub Actions, GitLab CI, CircleCI
  • Configuration Files — Docker Compose, Kubernetes, Ansible
  • Data Serialization — API responses, data exchange
  • Application Config — Settings, environment variables

📖 Syntax Guide

Basic Data Types

# Scalars
string: "Hello World"
number: 42
float: 3.14
boolean: true
null_value: null

# Unquoted strings (no special chars)
name: John Doe
city: New York

Lists (Arrays)

# Block style
fruits:
  - apple
  - banana
  - orange

# Flow style
colors: [red, green, blue]

# Mixed
mixed:
  - item1
  - item2
  - [nested, list]

Dictionaries (Objects)

# Basic object
person:
  name: John Doe
  age: 30
  city: New York

# Nested objects
company:
  name: Tech Corp
  address:
    street: 123 Main St
    city: Boston
    zip: "02101"

Multi-line Strings

# Literal (preserves newlines)
description: |
  This is a multi-line
  string that preserves
  line breaks.

# Folded (single line)
summary: >
  This text will be
  folded into a
  single line.

# Quotes for special characters
message: "Line 1\nLine 2"

Comments

# This is a comment
key: value  # Inline comment

# Multi-line comment block:
# Line 1
# Line 2
# Line 3

Anchors and Aliases

# Define anchor
defaults: &defaults
  timeout: 30
  retry: 3

# Reference anchor
production:
  <<: *defaults
  environment: prod

staging:
  <<: *defaults
  environment: staging

🔧 Using YAML in Code

JavaScript/Node.js

// Install: npm install js-yaml
const yaml = require('js-yaml');
const fs = require('fs');

// Parse YAML file
const config = yaml.load(fs.readFileSync('config.yml', 'utf8'));
console.log(config);

// Convert object to YAML
const obj = { name: 'John', age: 30 };
const yamlStr = yaml.dump(obj);
console.log(yamlStr);

Python

# Install: pip install pyyaml
import yaml

# Load YAML file
with open('config.yml', 'r') as file:
    config = yaml.safe_load(file)
    print(config)

# Dump to YAML
data = {'name': 'John', 'age': 30}
with open('output.yml', 'w') as file:
    yaml.dump(data, file)

Best Practices

  • Indentation — Use 2 spaces (never tabs)
  • Quotes — Use for strings with special characters
  • Comments — Document complex configurations
  • Anchors — Reuse common configurations
  • Validation — Use YAML linters

Common Pitfalls

  • ❌ Mixing tabs and spaces
  • ❌ Inconsistent indentation
  • ❌ Unquoted strings with colons
  • ❌ Missing spaces after colons
