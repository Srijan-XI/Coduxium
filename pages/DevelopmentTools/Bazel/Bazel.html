📚 Introduction

Bazel is a high-performance build system designed for large, multi-language repositories. It emphasizes reproducibility, hermeticity, and incremental builds for developer velocity and reliable CI pipelines.

Why Teams Use Bazel

  • Fast Incremental Builds - rebuild only changed targets
  • Reproducibility - consistent outputs across machines
  • Polyglot Support - Java, C++, Python, Go, JS, and more
  • Remote Cache/Execution - accelerate team and CI workflows

⚙️ Installation

# Windows (Chocolatey)
choco install bazel

# macOS
brew install bazelisk

# Linux (Bazelisk)
curl -L https://github.com/bazelbuild/bazelisk/releases/latest/download/bazelisk-linux-amd64 -o bazel
chmod +x bazel
sudo mv bazel /usr/local/bin/bazel

Tip: Bazelisk is commonly preferred because it auto-selects the Bazel version defined by your repository.

📖 User Guide

Minimal Example

# BUILD
cc_binary(
    name = "hello",
    srcs = ["hello.cc"],
)
# Build target
bazel build //:hello

# Run target
bazel run //:hello

# Run tests
bazel test //...

Typical Commands

  • bazel query for dependency graph exploration
  • bazel clean for cache cleanup when needed
  • bazel test --test_output=errors //... for debugging failing tests

🧠 Advanced Topics

Remote Cache in CI

build --remote_cache=https://your-cache-endpoint
build --remote_timeout=60
test --remote_cache=https://your-cache-endpoint

Best Practices

  • Keep BUILD targets small and composable
  • Prefer explicit dependencies to avoid hidden coupling
  • Use shared rule sets to standardize language builds
  • Pin external deps for deterministic outputs
