📚 Introduction

What is Postman?

Postman is a comprehensive API development platform that simplifies each step of building an API and streamlines collaboration. From design and testing to documentation and monitoring, Postman is the tool used by millions of developers and companies worldwide to accelerate API development.

Key Features

🔧 API Design — Design APIs with OpenAPI specifications
📡 Request Builder — Send HTTP requests with any method
📚 Collections — Organize and share API requests
🌍 Environments — Switch between dev/staging/prod
🧪 Testing — Automated tests with JavaScript
📊 Mock Servers — Simulate API responses
📝 Documentation — Auto-generated API docs
⚙️ Newman — CLI for CI/CD integration

Why Choose Postman?

  • Intuitive Interface: User-friendly GUI makes API testing accessible to everyone on your team.
  • Collaboration: Share collections, environments, and workspaces with your team.
  • Automation: Write tests in JavaScript and run them via Newman in CI/CD pipelines.
  • Comprehensive: Handle REST, GraphQL, WebSocket, gRPC, and SOAP APIs.
  • Ecosystem: Integrations with GitHub, GitLab, Slack, Jenkins, and more.
  • Free Tier: Generous free plan for individuals and small teams.

Use Cases

🧪 API Testing — Manual and automated API testing
🔍 API Exploration — Discover and understand APIs
⚙️ Automated Testing — Run tests in CI/CD pipelines
📝 Documentation — Generate and publish API docs
🏗️ Development — Design APIs before implementation
🤝 Collaboration — Share APIs across teams

Postman vs Alternatives

  • vs cURL: Visual interface, test automation, collaboration features
  • vs Insomnia: Larger ecosystem, better collaboration, more integrations
  • vs Thunder Client: Full-featured desktop app vs VS Code extension
  • vs HTTPie: GUI vs CLI focus, but both complement each other
⚠️ Important: Store sensitive data (API keys, tokens) in environment variables, never hardcode them in requests or collections that will be version-controlled.

Linux Installation

sudo snap install postman

Manual Installation

# Download and extract
wget https://dl.pstmn.io/download/latest/linux_64 -O postman-linux-x64.tar.gz
sudo tar -xzf postman-linux-x64.tar.gz -C /opt

sudo ln -s /opt/Postman/Postman /usr/bin/postman

Newman CLI Installation

Newman is the command-line companion for Postman that allows you to run collections in CI/CD pipelines.

# Install Newman globally via npm
npm install -g newman

# Verify installation
newman --version

# Install Newman HTML reporter (optional but recommended)
npm install -g newman-reporter-htmlextra

# Install Newman JUnit reporter
npm install -g newman-reporter-junit

📖 User Guide

Learn how to create requests, build collections, write tests, and automate API testing.

Making HTTP Requests

Create a Simple Request

1 Click "New" or "+" to create a new request
2 Select HTTP method (GET, POST, PUT, DELETE, etc.)
3 Enter URL (e.g., https://api.example.com/users)
4 Add headers, body, or parameters if needed
5 Click "Send"
6 View response (body, headers, status)

Request Types Examples

# GET Request
GET https://jsonplaceholder.typicode.com/posts/1

# POST Request with JSON body
POST https://jsonplaceholder.typicode.com/posts
Headers:
  Content-Type: application/json
Body (raw JSON):
{
  "title": "My Post",
  "body": "This is the content",
  "userId": 1
}

# PUT Request (update)
PUT https://jsonplaceholder.typicode.com/posts/1
Body:
{
  "id": 1,
  "title": "Updated Title",
  "body": "Updated content",
  "userId": 1
}

# DELETE Request
DELETE https://jsonplaceholder.typicode.com/posts/1

# Query Parameters
GET https://api.example.com/users?page=2&limit=10

Writing Tests

Basic Tests

// Tests tab in request

// Check status code
pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});

// Check response time
pm.test("Response time is less than 200ms", function () {
    pm.expect(pm.response.responseTime).to.be.below(200);
});

// Check JSON response
pm.test("Response has JSON body", function () {
    pm.response.to.be.json;
});

// Check specific field
pm.test("User has correct email", function () {
    const jsonData = pm.response.json();
    pm.expect(jsonData.email).to.eql("user@example.com");
});

// Check array length
pm.test("Returns 10 users", function () {
    const jsonData = pm.response.json();
    pm.expect(jsonData).to.have.lengthOf(10);
});

// Check header exists
pm.test("Content-Type header is present", function () {
    pm.response.to.have.header("Content-Type");
});

// Multiple assertions
pm.test("User object is valid", function () {
    const user = pm.response.json();
    pm.expect(user).to.have.property("id");
    pm.expect(user).to.have.property("name");
    pm.expect(user.name).to.be.a("string");
    pm.expect(user.id).to.be.a("number");
});

Advanced Tests

// Save response data to environment
pm.test("Save user ID", function () {
    const jsonData = pm.response.json();
    pm.environment.set("userId", jsonData.id);
    pm.environment.set("authToken", jsonData.token);
});

// Conditional tests
if (pm.response.code === 200) {
    pm.test("Successful response has data", function () {
        const jsonData = pm.response.json();
        pm.expect(jsonData).to.have.property("data");
    });
}

// Schema validation
const schema = {
    type: "object",
    required: ["id", "email", "name"],
    properties: {
        id: { type: "number" },
        email: { type: "string", format: "email" },
        name: { type: "string", minLength: 1 }
    }
};

pm.test("Response matches schema", function () {
    pm.response.to.have.jsonSchema(schema);
});

// Array validation
pm.test("All users have valid emails", function () {
    const users = pm.response.json();
    users.forEach(user => {
        pm.expect(user.email).to.match(/^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$/);
    });
});

Pre-request Scripts

// Generate timestamp
pm.environment.set("timestamp", Date.now());

// Generate random data
const randomEmail = `user${Math.floor(Math.random() * 10000)}@example.com`;
pm.environment.set("randomEmail", randomEmail);

// Set authorization header
const token = pm.environment.get("authToken");
pm.request.headers.add({
    key: "Authorization",
    value: `Bearer ${token}`
});

// Generate UUID
const { v4: uuidv4 } = require('uuid');
pm.environment.set("requestId", uuidv4());

// Hash password (example)
const CryptoJS = require('crypto-js');
const password = pm.environment.get("password");
const hashedPassword = CryptoJS.SHA256(password).toString();
pm.environment.set("hashedPassword", hashedPassword);

// Make a pre-request API call
pm.sendRequest({
    url: pm.environment.get("baseUrl") + "/auth/token",
    method: "POST",
    header: {
        "Content-Type": "application/json"
    },
    body: {
        mode: "raw",
        raw: JSON.stringify({
            username: pm.environment.get("username"),
            password: pm.environment.get("password")
        })
    }
}, function (err, res) {
    if (!err) {
        const token = res.json().access_token;
        pm.environment.set("authToken", token);
    }
});

Newman CLI

Basic Newman Usage

# Run collection
newman run my-api.postman_collection.json

# Run with environment
newman run my-api.postman_collection.json \
  -e local.postman_environment.json

# Run with specific iteration count
newman run my-api.postman_collection.json \
  --iteration-count 5

# Run with delay between requests
newman run my-api.postman_collection.json \
  --delay-request 1000

# Run specific folder
newman run my-api.postman_collection.json \
  --folder "User Tests"

Newman with Reporters

# Generate HTML report
newman run my-api.postman_collection.json \
  -e local.postman_environment.json \
  -r htmlextra \
  --reporter-htmlextra-export report.html

# Generate JUnit report (for CI/CD)
newman run my-api.postman_collection.json \
  -e prod.postman_environment.json \
  -r junit \
  --reporter-junit-export results.xml

# Multiple reporters
newman run my-api.postman_collection.json \
  -r cli,json,htmlextra \
  --reporter-json-export output.json \
  --reporter-htmlextra-export report.html

# Run in CI/CD (exit on failure)
newman run my-api.postman_collection.json \
  -e prod.postman_environment.json \
  --bail \
  --suppress-exit-code

Newman in CI/CD

# GitHub Actions example
name: API Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '20'
      - run: npm install -g newman newman-reporter-htmlextra
      - run: |
          newman run api-tests.postman_collection.json \
            -e production.postman_environment.json \
            -r htmlextra,junit \
            --reporter-htmlextra-export test-report.html \
            --reporter-junit-export results.xml
      - uses: actions/upload-artifact@v3
        if: always()
        with:
          name: test-reports
          path: |
            test-report.html
            results.xml
