🦜🔗 LangChain

A framework for developing applications powered by large language models with chaining and memory.

Overview

LLM AI Agents Python/TypeScript Chains

LangChain is a framework for developing applications powered by language models. It enables applications to be context-aware, connect LLMs to sources of context, and use reasoning to determine actions based on that context.

Key Features

  • Model I/O — Manage prompts and interact with LLMs
  • Chains — Sequence multiple components together
  • Agents — Use LLMs to choose which actions to take
  • Memory — Persist state between chain/agent calls
  • Document Loaders — Load data from various sources
  • Vector Stores — Semantic search integration
  • Retrievers — Combine docs with language models
  • Callbacks — Hook into any LLM application

Common Use Cases

  • Question-answering systems over documents
  • Chatbots with memory and context
  • AI agents that can use tools
  • Document summarization
  • Code generation and analysis
  • Data extraction and transformation
  • Semantic search applications
  • Multi-step reasoning tasks

LangChain Ecosystem

  • LangChain — Core Python/TypeScript framework
  • LangSmith — Debugging and monitoring platform
  • LangServe — Deploy chains as REST APIs
  • LangGraph — Build stateful multi-actor applications
  • LangChain Hub — Share and discover prompts
  • Templates — Reference architectures

Getting Started

Install LangChain:

pip install langchain langchain-community langchain-openai

Simple chain example:

from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.schema.output_parser import StrOutputParser

# Create a chain
prompt = ChatPromptTemplate.from_template("Tell me a joke about {topic}")
model = ChatOpenAI()
output_parser = StrOutputParser()

chain = prompt | model | output_parser

# Run the chain
result = chain.invoke({"topic": "programming"})

RAG example:

from langchain.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.vectorstores import FAISS
from langchain.embeddings import OpenAIEmbeddings
from langchain.chains import RetrievalQA

# Load and split documents
loader = TextLoader("data.txt")
documents = loader.load()
text_splitter = RecursiveCharacterTextSplitter()
texts = text_splitter.split_documents(documents)

# Create vector store
embeddings = OpenAIEmbeddings()
vectorstore = FAISS.from_documents(texts, embeddings)

# Create QA chain
qa_chain = RetrievalQA.from_chain_type(
    llm=ChatOpenAI(),
    retriever=vectorstore.as_retriever()
)

result = qa_chain.run("Your question here")

Best Practices

  • Use LCEL (LangChain Expression Language) for chains
  • Implement proper error handling
  • Monitor costs and token usage
  • Use async operations for better performance
  • Leverage caching to reduce API calls
  • Test chains with LangSmith
  • Use streaming for better UX
  • Implement rate limiting
