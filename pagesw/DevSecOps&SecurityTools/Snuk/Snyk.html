🛡️ Snyk

Developer-first security platform for finding and automatically fixing vulnerabilities in code, dependencies, containers, and IaC.

Overview

SECURITY DEVSECOPS DEPENDENCIES DEVELOPER-FIRST

Snyk is a developer security platform that helps find and fix vulnerabilities across your entire technology stack: open source dependencies, container images, Infrastructure as Code (IaC), and application code.

Key Products

  • Snyk Open Source — Find and fix vulnerabilities in dependencies
  • Snyk Code — SAST (Static Application Security Testing)
  • Snyk Container — Scan container images and Kubernetes workloads
  • Snyk Infrastructure as Code — Scan Terraform, CloudFormation, Kubernetes YAML
  • Snyk Cloud — Runtime security and cloud posture management

Snyk CLI Installation

# macOS
brew install snyk/tap/snyk

# Linux/macOS (npm)
npm install -g snyk

# Windows (Scoop)
scoop bucket add snyk https://github.com/snyk/scoop-snyk
scoop install snyk

# Docker
docker run --rm -it -v $(pwd):/app snyk/snyk:linux test

# Authenticate
snyk auth

# Verify installation
snyk --version

Testing Dependencies

# Test project for vulnerabilities
snyk test

# Test specific package manager
snyk test --file=package.json
snyk test --file=pom.xml
snyk test --file=requirements.txt
snyk test --file=Gemfile.lock

# Test and monitor
snyk test && snyk monitor

# Test with JSON output
snyk test --json

# Test with severity threshold
snyk test --severity-threshold=high

# Ignore specific vulnerabilities
snyk ignore --id=SNYK-JS-LODASH-590103

# Fix vulnerabilities automatically
snyk fix

Container Image Scanning

# Test Docker image
snyk container test nginx:latest

# Test with Dockerfile
snyk container test nginx:latest --file=Dockerfile

# Monitor image
snyk container monitor nginx:latest

# Test with specific platform
snyk container test nginx:latest --platform=linux/amd64

# Generate SBOM
snyk container test nginx:latest --print-deps --json > sbom.json

# Test local image
docker build -t myapp:latest .
snyk container test myapp:latest

# Exclude base image vulnerabilities
snyk container test myapp:latest --exclude-base-image-vulns

Infrastructure as Code Scanning

# Test Terraform files
snyk iac test terraform/

# Test Kubernetes manifests
snyk iac test k8s-deployment.yaml

# Test CloudFormation
snyk iac test cloudformation-template.yaml

# Test Helm charts
snyk iac test helm/

# Test with specific rules
snyk iac test --rules=custom-rules.tar.gz

# Generate HTML report
snyk iac test --report --report-format=html > report.html

# Test with severity threshold
snyk iac test --severity-threshold=medium

Code Scanning (SAST)

# Test source code
snyk code test

# Test specific file
snyk code test src/app.js

# Test with JSON output
snyk code test --json

# Test and get detailed report
snyk code test --sarif > results.sarif

# Ignore specific issues
snyk ignore --id=javascript/NoHardcodedCredentials

GitHub Actions Integration

name: Snyk Security Scan

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  snyk:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Run Snyk to check for vulnerabilities
        uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
        with:
          args: --severity-threshold=high
      
      - name: Upload result to GitHub Code Scanning
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: snyk.sarif

  snyk-container:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Build Docker image
        run: docker build -t myapp:${{ github.sha }} .
      
      - name: Run Snyk Container scan
        uses: snyk/actions/docker@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
        with:
          image: myapp:${{ github.sha }}
          args: --severity-threshold=critical

  snyk-iac:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Run Snyk IaC scan
        uses: snyk/actions/iac@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
        with:
          file: terraform/

GitLab CI Integration

stages:
  - security

snyk-scan:
  stage: security
  image: snyk/snyk:node
  script:
    - snyk auth $SNYK_TOKEN
    - snyk test --severity-threshold=high
    - snyk monitor
  allow_failure: true

snyk-container:
  stage: security
  image: snyk/snyk:docker
  services:
    - docker:dind
  script:
    - docker build -t $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA .
    - snyk auth $SNYK_TOKEN
    - snyk container test $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA
  allow_failure: false

snyk-iac:
  stage: security
  image: snyk/snyk:linux
  script:
    - snyk auth $SNYK_TOKEN
    - snyk iac test terraform/ --severity-threshold=medium
  allow_failure: true

Jenkins Integration

pipeline {
    agent any
    
    environment {
        SNYK_TOKEN = credentials('snyk-api-token')
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Build') {
            steps {
                sh 'npm install'
            }
        }
        
        stage('Snyk Security Scan') {
            steps {
                script {
                    snykSecurity(
                        snykInstallation: 'snyk-latest',
                        snykTokenId: 'snyk-api-token',
                        severity: 'high',
                        failOnIssues: true
                    )
                }
            }
        }
        
        stage('Snyk Monitor') {
            steps {
                sh 'snyk monitor --project-name=my-app'
            }
        }
    }
}

Snyk API Example

import requests

# Configuration
SNYK_TOKEN = "your-snyk-token"
ORG_ID = "your-org-id"
BASE_URL = "https://api.snyk.io/v1"

headers = {
    "Authorization": f"token {SNYK_TOKEN}",
    "Content-Type": "application/json"
}

# List projects
response = requests.get(
    f"{BASE_URL}/org/{ORG_ID}/projects",
    headers=headers
)
projects = response.json()

for project in projects['projects']:
    print(f"Project: {project['name']}")
    print(f"Issues: {project['issueCountsBySeverity']}")

# Get vulnerabilities for a project
project_id = "project-id"
response = requests.post(
    f"{BASE_URL}/org/{ORG_ID}/project/{project_id}/issues",
    headers=headers,
    json={"filters": {"severities": ["high", "critical"]}}
)
issues = response.json()
print(f"Critical/High issues: {len(issues['issues'])}")
