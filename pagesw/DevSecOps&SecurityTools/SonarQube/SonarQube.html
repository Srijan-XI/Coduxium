🔍 SonarQube

Leading platform for continuous inspection of code quality and security vulnerabilities.

Overview

CODE QUALITY SECURITY STATIC ANALYSIS DEVSECOPS

SonarQube is an open-source platform for continuous inspection of code quality to perform automatic reviews with static analysis to detect bugs, code smells, and security vulnerabilities across 30+ programming languages.

Key Features

  • Multi-Language Support — Java, C#, Python, JavaScript, TypeScript, Go, and more
  • Code Quality Metrics — Bugs, code smells, technical debt
  • Security Vulnerabilities — OWASP Top 10, CWE coverage
  • Quality Gates — Enforce quality standards before merge
  • Code Coverage — Track test coverage metrics
  • Pull Request Decoration — GitHub, GitLab, Bitbucket integration
  • Custom Rules — Define organization-specific rules
  • Technical Debt — Measure and track code debt

Installation - Docker

# Pull SonarQube image
docker pull sonarqube:lts-community

# Run SonarQube
docker run -d --name sonarqube \
  -p 9000:9000 \
  -e SONAR_ES_BOOTSTRAP_CHECKS_DISABLE=true \
  sonarqube:lts-community

# With PostgreSQL database
docker network create sonarnet

docker run -d --name sonarqube-db \
  --network sonarnet \
  -e POSTGRES_USER=sonar \
  -e POSTGRES_PASSWORD=sonar \
  -e POSTGRES_DB=sonarqube \
  postgres:15

docker run -d --name sonarqube \
  --network sonarnet \
  -p 9000:9000 \
  -e SONAR_JDBC_URL=jdbc:postgresql://sonarqube-db:5432/sonarqube \
  -e SONAR_JDBC_USERNAME=sonar \
  -e SONAR_JDBC_PASSWORD=sonar \
  sonarqube:lts-community

# Access at http://localhost:9000
# Default credentials: admin/admin

SonarScanner Setup

# Install SonarScanner CLI
# macOS
brew install sonar-scanner

# Linux (manual)
wget https://binaries.sonarsource.com/Distribution/sonar-scanner-cli/sonar-scanner-cli-5.0.1.3006-linux.zip
unzip sonar-scanner-cli-5.0.1.3006-linux.zip
export PATH=$PATH:/path/to/sonar-scanner/bin

# Windows
# Download from: https://docs.sonarqube.org/latest/analysis/scan/sonarscanner/

# Verify installation
sonar-scanner --version

# sonar-project.properties
sonar.projectKey=my-project
sonar.projectName=My Project
sonar.projectVersion=1.0
sonar.sources=src
sonar.host.url=http://localhost:9000
sonar.login=YOUR_TOKEN

# Run scan
sonar-scanner

Maven Integration

<!-- Add to pom.xml -->
<properties>
    <sonar.host.url>http://localhost:9000</sonar.host.url>
    <sonar.login>YOUR_TOKEN</sonar.login>
</properties>

<!-- Run analysis -->
mvn clean verify sonar:sonar

<!-- With parameters -->
mvn clean verify sonar:sonar \
  -Dsonar.projectKey=my-project \
  -Dsonar.host.url=http://localhost:9000 \
  -Dsonar.login=YOUR_TOKEN

GitHub Actions Integration

name: SonarQube Analysis

on:
  push:
    branches: [main, develop]
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  sonarqube:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0  # Shallow clones disabled for better analysis

      - name: Set up JDK 17
        uses: actions/setup-java@v3
        with:
          java-version: 17
          distribution: 'temurin'

      - name: Cache SonarQube packages
        uses: actions/cache@v3
        with:
          path: ~/.sonar/cache
          key: ${{ runner.os }}-sonar
          restore-keys: ${{ runner.os }}-sonar

      - name: Build and analyze
        env:
          SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
          SONAR_HOST_URL: ${{ secrets.SONAR_HOST_URL }}
        run: mvn -B verify org.sonarsource.scanner.maven:sonar-maven-plugin:sonar \
          -Dsonar.projectKey=my-project

GitLab CI Integration

stages:
  - test
  - sonarqube

sonarqube-check:
  stage: sonarqube
  image: maven:3.9-eclipse-temurin-17
  variables:
    SONAR_USER_HOME: "${CI_PROJECT_DIR}/.sonar"
    GIT_DEPTH: "0"
  cache:
    key: "${CI_JOB_NAME}"
    paths:
      - .sonar/cache
  script:
    - mvn verify sonar:sonar 
      -Dsonar.projectKey=$CI_PROJECT_NAME 
      -Dsonar.host.url=$SONAR_HOST_URL 
      -Dsonar.login=$SONAR_TOKEN
  allow_failure: true
  only:
    - merge_requests
    - main
    - develop

Quality Gate Configuration

# Quality Gate conditions (configured in SonarQube UI)
# - Coverage < 80%: FAILED
# - Duplicated Lines > 3%: FAILED
# - Maintainability Rating worse than A: FAILED
# - Reliability Rating worse than A: FAILED
# - Security Rating worse than A: FAILED
# - Security Hotspots Reviewed < 100%: FAILED

# Check Quality Gate status in pipeline
sonar-scanner \
  -Dsonar.qualitygate.wait=true \
  -Dsonar.qualitygate.timeout=300

# Exit code 0 if passed, 1 if failed

JavaScript/TypeScript Analysis

# sonar-project.properties for Node.js
sonar.projectKey=my-js-project
sonar.projectName=My JavaScript Project
sonar.sources=src
sonar.tests=test
sonar.javascript.lcov.reportPaths=coverage/lcov.info
sonar.testExecutionReportPaths=test-report.xml

# Run tests with coverage
npm test -- --coverage

# Run SonarScanner
sonar-scanner

# TypeScript specific
sonar.typescript.tsconfigPath=tsconfig.json

Python Analysis

# sonar-project.properties for Python
sonar.projectKey=my-python-project
sonar.projectName=My Python Project
sonar.sources=src
sonar.tests=tests
sonar.python.coverage.reportPaths=coverage.xml
sonar.python.version=3.11

# Generate coverage with pytest
pytest --cov=src --cov-report=xml

# Run SonarScanner
sonar-scanner
