🔒 Aqua Security

Cloud-native application protection platform (CNAPP) for securing containers, Kubernetes, and serverless.

Overview

DEVSECOPS CONTAINERS KUBERNETES CNAPP

Aqua Security provides end-to-end security for cloud-native applications, from development to production. It offers vulnerability scanning, runtime protection, compliance, and security policy enforcement for containers, Kubernetes, and serverless workloads.

Key Features

  • Image Scanning — Vulnerability detection in container images
  • Runtime Protection — Detect and prevent threats at runtime
  • Kubernetes Security — RBAC, network policies, admission control
  • Compliance — CIS benchmarks, PCI-DSS, HIPAA compliance
  • Serverless Security — AWS Lambda, Azure Functions protection
  • SBOM Generation — Software Bill of Materials
  • Secrets Management — Detect exposed secrets in images
  • CI/CD Integration — Shift-left security in pipelines

Aqua Trivy - Open Source Scanner

# Install Trivy
# macOS
brew install trivy

# Linux
wget -qO - https://aquasecurity.github.io/trivy-repo/deb/public.key | sudo apt-key add -
echo "deb https://aquasecurity.github.io/trivy-repo/deb $(lsb_release -sc) main" | sudo tee -a /etc/apt/sources.list.d/trivy.list
sudo apt-get update
sudo apt-get install trivy

# Container
docker run --rm -v /var/run/docker.sock:/var/run/docker.sock aquasec/trivy image alpine:3.18

# Verify installation
trivy --version

Scanning Container Images

# Scan Docker image
trivy image nginx:latest

# Scan with severity filter
trivy image --severity HIGH,CRITICAL nginx:latest

# Output as JSON
trivy image --format json --output results.json nginx:latest

# Scan specific vulnerabilities
trivy image --vuln-type os,library nginx:latest

# Scan local image
docker build -t myapp:latest .
trivy image myapp:latest

# Scan image from tar file
trivy image --input image.tar

# Ignore unfixed vulnerabilities
trivy image --ignore-unfixed nginx:latest

# Exit on severity
trivy image --exit-code 1 --severity CRITICAL nginx:latest

Filesystem Scanning

# Scan filesystem
trivy fs /path/to/project

# Scan for secrets
trivy fs --scanners secret /path/to/project

# Scan for misconfigurations
trivy fs --scanners config /path/to/project

# Scan Terraform
trivy config terraform/

# Scan Kubernetes manifests
trivy config k8s-manifests/

# Scan Docker Compose
trivy config docker-compose.yml

# Generate SBOM
trivy image --format cyclonedx --output sbom.json nginx:latest

CI/CD Integration - GitHub Actions

name: Security Scan

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  trivy-scan:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Build Docker image
        run: docker build -t myapp:${{ github.sha }} .

      - name: Run Trivy vulnerability scanner
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: 'myapp:${{ github.sha }}'
          format: 'sarif'
          output: 'trivy-results.sarif'
          severity: 'CRITICAL,HIGH'

      - name: Upload Trivy results to GitHub Security
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: 'trivy-results.sarif'

      - name: Fail on high vulnerabilities
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: 'myapp:${{ github.sha }}'
          exit-code: '1'
          severity: 'CRITICAL,HIGH'

GitLab CI Integration

stages:
  - build
  - security

build:
  stage: build
  script:
    - docker build -t $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA .
    - docker push $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA

trivy-scan:
  stage: security
  image: aquasec/trivy:latest
  script:
    - trivy image --exit-code 0 --no-progress --format json -o trivy-report.json $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA
    - trivy image --exit-code 1 --severity CRITICAL --no-progress $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA
  artifacts:
    reports:
      container_scanning: trivy-report.json
  allow_failure: false

Aqua Policy Example

# aqua-policy.yaml - Runtime policy
apiVersion: aquasecurity.github.io/v1alpha1
kind: RuntimePolicy
metadata:
  name: block-privilege-escalation
spec:
  scope:
    - workload: deployment
      namespace: production
  controls:
    - name: prevent-privilege-escalation
      action: block
      settings:
        allowPrivilegeEscalation: false
    - name: block-root-user
      action: block
      settings:
        runAsNonRoot: true
    - name: readonly-root-filesystem
      action: alert
      settings:
        readOnlyRootFilesystem: true

Best Practices

  • Scan images in CI/CD before deployment
  • Use minimal base images (Alpine, Distroless)
  • Keep images updated regularly
  • Implement image signing and verification
  • Use private registries with access control
  • Enable runtime protection in production
  • Monitor for compliance violations
  • Automate remediation workflows
  • Regular security audits and reports

Aqua Platform Components

  • Aqua Server — Central management console
  • Aqua Gateway — Communication hub for enforcers
  • Aqua Enforcer — Runtime protection agent
  • Aqua Scanner — Image scanning service
  • Trivy — Open-source vulnerability scanner
  • Tracee — Runtime security and forensics
