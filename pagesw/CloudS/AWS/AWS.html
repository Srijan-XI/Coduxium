☁️ Amazon Web Services (AWS)

World's most comprehensive and widely adopted cloud platform with over 200 services.

Overview

CLOUD INFRASTRUCTURE GLOBAL SCALABLE

Amazon Web Services (AWS) is a comprehensive cloud computing platform offering over 200 fully featured services from data centers globally. It powers millions of customers including startups, enterprises, and government agencies.

Core Services

  • EC2 (Elastic Compute Cloud) — Virtual servers in the cloud
  • S3 (Simple Storage Service) — Object storage with 99.999999999% durability
  • RDS (Relational Database Service) — Managed relational databases
  • Lambda — Serverless compute service
  • VPC (Virtual Private Cloud) — Isolated cloud network
  • CloudFront — Content delivery network (CDN)
  • DynamoDB — NoSQL database service
  • ECS/EKS — Container orchestration (Docker/Kubernetes)

AWS CLI Installation

# Windows (using MSI installer)
# Download from: https://aws.amazon.com/cli/

# macOS
brew install awscli

# Linux
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Verify installation
aws --version

# Configure AWS CLI
aws configure
# AWS Access Key ID: YOUR_ACCESS_KEY
# AWS Secret Access Key: YOUR_SECRET_KEY
# Default region name: us-east-1
# Default output format: json

EC2 Instance Management

# List all EC2 instances
aws ec2 describe-instances

# Launch a new instance
aws ec2 run-instances \
    --image-id ami-0c55b159cbfafe1f0 \
    --instance-type t2.micro \
    --key-name MyKeyPair \
    --security-groups my-sg \
    --count 1

# Stop an instance
aws ec2 stop-instances --instance-ids i-1234567890abcdef0

# Start an instance
aws ec2 start-instances --instance-ids i-1234567890abcdef0

# Terminate an instance
aws ec2 terminate-instances --instance-ids i-1234567890abcdef0

S3 Operations

# Create a bucket
aws s3 mb s3://my-bucket-name

# List all buckets
aws s3 ls

# Upload a file
aws s3 cp myfile.txt s3://my-bucket-name/

# Download a file
aws s3 cp s3://my-bucket-name/myfile.txt ./

# Sync directory to S3
aws s3 sync ./local-folder s3://my-bucket-name/folder/

# Delete a file
aws s3 rm s3://my-bucket-name/myfile.txt

# Delete bucket
aws s3 rb s3://my-bucket-name --force

Lambda Function Example

# Node.js Lambda function
exports.handler = async (event) => {
    console.log('Event:', JSON.stringify(event, null, 2));
    
    const response = {
        statusCode: 200,
        body: JSON.stringify({
            message: 'Hello from Lambda!',
            input: event
        })
    };
    
    return response;
};

# Python Lambda function
import json

def lambda_handler(event, context):
    print("Event:", json.dumps(event))
    
    return {
        'statusCode': 200,
        'body': json.dumps({
            'message': 'Hello from Lambda!',
            'input': event
        })
    }

# Deploy Lambda with AWS CLI
aws lambda create-function \
    --function-name MyFunction \
    --runtime nodejs18.x \
    --role arn:aws:iam::123456789012:role/lambda-role \
    --handler index.handler \
    --zip-file fileb://function.zip

AWS SDK Example (Python - Boto3)

import boto3

# Create S3 client
s3 = boto3.client('s3')

# Upload file
s3.upload_file('local-file.txt', 'my-bucket', 'remote-file.txt')

# Download file
s3.download_file('my-bucket', 'remote-file.txt', 'downloaded-file.txt')

# List objects
response = s3.list_objects_v2(Bucket='my-bucket')
for obj in response.get('Contents', []):
    print(obj['Key'])

# EC2 operations
ec2 = boto3.resource('ec2')

# Launch instance
instances = ec2.create_instances(
    ImageId='ami-0c55b159cbfafe1f0',
    MinCount=1,
    MaxCount=1,
    InstanceType='t2.micro',
    KeyName='MyKeyPair'
)

# DynamoDB operations
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('Users')

# Put item
table.put_item(
    Item={
        'userId': '123',
        'name': 'John Doe',
        'email': 'john@example.com'
    }
)

# Get item
response = table.get_item(Key={'userId': '123'})
print(response['Item'])

CloudFormation Template Example

AWSTemplateFormatVersion: '2010-09-09'
Description: 'Simple EC2 instance with S3 bucket'

Resources:
  MyS3Bucket:
    Type: AWS::S3::Bucket
    Properties:
      BucketName: my-application-bucket
      VersioningConfiguration:
        Status: Enabled

  MyEC2Instance:
    Type: AWS::EC2::Instance
    Properties:
      InstanceType: t2.micro
      ImageId: ami-0c55b159cbfafe1f0
      KeyName: MyKeyPair
      SecurityGroups:
        - !Ref MySecurityGroup

  MySecurityGroup:
    Type: AWS::EC2::SecurityGroup
    Properties:
      GroupDescription: Allow SSH and HTTP
      SecurityGroupIngress:
        - IpProtocol: tcp
          FromPort: 22
          ToPort: 22
          CidrIp: 0.0.0.0/0
        - IpProtocol: tcp
          FromPort: 80
          ToPort: 80
          CidrIp: 0.0.0.0/0

Outputs:
  InstanceId:
    Description: Instance ID
    Value: !Ref MyEC2Instance
  BucketName:
    Description: S3 Bucket Name
    Value: !Ref MyS3Bucket
