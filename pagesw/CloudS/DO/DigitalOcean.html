Developer-friendly cloud computing platform with simplicity, transparency, and predictable pricing.

Overview

CLOUD DEVELOPER SIMPLE AFFORDABLE

DigitalOcean is a cloud infrastructure provider focused on simplifying cloud computing for developers, startups, and SMBs. Known for its simple pricing, excellent documentation, and developer-centric approach.

Core Products

  • Droplets — Virtual machines (VPS)
  • Kubernetes (DOKS) — Managed Kubernetes service
  • App Platform — PaaS for deploying apps from Git
  • Spaces — S3-compatible object storage
  • Managed Databases — PostgreSQL, MySQL, Redis, MongoDB
  • Load Balancers — High availability traffic distribution
  • Volumes — Block storage for Droplets
  • VPC — Virtual Private Cloud networking

doctl CLI Installation

# macOS
brew install doctl

# Linux (snap)
sudo snap install doctl

# Windows (scoop)
scoop install doctl

# Manual installation
# Download from: https://github.com/digitalocean/doctl/releases

# Authenticate
doctl auth init
# Enter your API token

# Verify authentication
doctl account get

# List regions
doctl compute region list

Droplet Management

# Create Droplet
doctl compute droplet create my-droplet \
    --region nyc3 \
    --size s-1vcpu-1gb \
    --image ubuntu-22-04-x64 \
    --ssh-keys YOUR_SSH_KEY_ID

# List Droplets
doctl compute droplet list

# Get Droplet info
doctl compute droplet get DROPLET_ID

# Power on/off
doctl compute droplet-action power-on DROPLET_ID
doctl compute droplet-action power-off DROPLET_ID

# Reboot Droplet
doctl compute droplet-action reboot DROPLET_ID

# Resize Droplet
doctl compute droplet-action resize DROPLET_ID --size s-2vcpu-4gb

# Delete Droplet
doctl compute droplet delete DROPLET_ID

# SSH into Droplet
doctl compute ssh DROPLET_ID

Spaces (Object Storage) Operations

# Install s3cmd for Spaces
sudo apt-get install s3cmd  # Linux
brew install s3cmd          # macOS

# Configure s3cmd for Spaces
s3cmd --configure
# Access Key: YOUR_ACCESS_KEY
# Secret Key: YOUR_SECRET_KEY
# S3 Endpoint: nyc3.digitaloceanspaces.com

# Create Space (bucket)
s3cmd mb s3://my-space-name

# List Spaces
s3cmd ls

# Upload file
s3cmd put local-file.txt s3://my-space-name/

# Download file
s3cmd get s3://my-space-name/file.txt ./

# Sync directory
s3cmd sync ./local-folder/ s3://my-space-name/folder/

# List objects
s3cmd ls s3://my-space-name/

# Delete object
s3cmd del s3://my-space-name/file.txt

# Make object public
s3cmd setacl s3://my-space-name/file.txt --acl-public

Kubernetes (DOKS)

# Create Kubernetes cluster
doctl kubernetes cluster create my-cluster \
    --region nyc3 \
    --version 1.28.2-do.0 \
    --node-pool "name=worker-pool;size=s-2vcpu-4gb;count=3"

# List clusters
doctl kubernetes cluster list

# Get kubeconfig
doctl kubernetes cluster kubeconfig save my-cluster

# Scale node pool
doctl kubernetes cluster node-pool update my-cluster worker-pool \
    --count 5

# Upgrade cluster
doctl kubernetes cluster upgrade my-cluster --version 1.29.0-do.0

# Delete cluster
doctl kubernetes cluster delete my-cluster

App Platform Deployment

# app.yaml - App Platform spec
name: my-app
services:
  - name: web
    github:
      repo: username/my-repo
      branch: main
      deploy_on_push: true
    build_command: npm install
    run_command: npm start
    environment_slug: node-js
    instance_size_slug: basic-xxs
    instance_count: 1
    http_port: 3000
    routes:
      - path: /
    envs:
      - key: DATABASE_URL
        scope: RUN_TIME
        value: ${db.DATABASE_URL}

databases:
  - name: db
    engine: PG
    version: "14"

# Deploy app
doctl apps create --spec app.yaml

# List apps
doctl apps list

# Get app info
doctl apps get APP_ID

# Update app
doctl apps update APP_ID --spec app.yaml

# Delete app
doctl apps delete APP_ID

DigitalOcean API Example (Python)

import requests

# API Configuration
API_TOKEN = "your-api-token"
BASE_URL = "https://api.digitalocean.com/v2"
headers = {
    "Authorization": f"Bearer {API_TOKEN}",
    "Content-Type": "application/json"
}

# List Droplets
response = requests.get(f"{BASE_URL}/droplets", headers=headers)
droplets = response.json()['droplets']
for droplet in droplets:
    print(f"Name: {droplet['name']}, IP: {droplet['networks']['v4'][0]['ip_address']}")

# Create Droplet
droplet_data = {
    "name": "my-droplet",
    "region": "nyc3",
    "size": "s-1vcpu-1gb",
    "image": "ubuntu-22-04-x64",
    "ssh_keys": ["YOUR_SSH_KEY_ID"],
    "backups": False,
    "ipv6": True,
    "monitoring": True
}
response = requests.post(f"{BASE_URL}/droplets", json=droplet_data, headers=headers)
print(response.json())

# Delete Droplet
droplet_id = "12345678"
response = requests.delete(f"{BASE_URL}/droplets/{droplet_id}", headers=headers)
print(f"Deleted: {response.status_code == 204}")

Managed Database Example

# Create managed PostgreSQL database
doctl databases create my-postgres-db \
    --engine pg \
    --region nyc3 \
    --size db-s-1vcpu-1gb \
    --version 14

# List databases
doctl databases list

# Get connection details
doctl databases connection my-postgres-db

# Create database user
doctl databases user create my-postgres-db new-user

# Create new database
doctl databases db create my-postgres-db new-database

# Get database info
doctl databases get my-postgres-db

# Resize database
doctl databases resize my-postgres-db --size db-s-2vcpu-4gb

# Connection string format
postgresql://username:password@host:port/database?sslmode=require

  • Use SSH keys instead of passwords

  • Tag resources for better organization
  • Use Spaces for static content and CDN
  • Monitor resource usage with built-in metrics
