📚 Introduction

What is Docker?

Docker is an open-source platform that automates the deployment of applications inside lightweight, portable containers. Containers package an application with all of its dependencies, ensuring it runs consistently across different computing environments - from your laptop to production servers.

Key Features

🐳 Containerization — Lightweight, standalone executable packages
📦 Image Layering — Efficient storage with copy-on-write file systems
🔄 Portability — Run anywhere Docker is installed
⚡ Fast Startup — Containers start in seconds
🔒 Isolation — Process and filesystem isolation
📊 Resource Efficient — Less overhead than VMs
🎯 Docker Compose — Multi-container orchestration
🌐 Docker Hub — Public registry with millions of images

Why Choose Docker?

  • Consistency: Eliminate "works on my machine" problems by ensuring identical environments everywhere.
  • Rapid Deployment: Build once, deploy anywhere. Containers start in seconds, making scaling instant.
  • Isolation: Run multiple versions of the same application or different applications without conflicts.
  • Efficiency: Containers sh are the host OS kernel, using fewer resources than traditional VMs.
  • Microservices: Perfect for microservice architectures where each service runs in its own container.
  • CI/CD Integration: Seamlessly integrates with Jenkins, GitLab CI, GitHub Actions, and other CI/CD tools.

Use Cases

🚀 Microservices — Deploy and scale services independently
🔄 CI/CD — Consistent build and test environments
☁️ Cloud Migration — Easy lift-and-shift to cloud platforms
💻 Development — Standardized dev environments for teams
📊 Data Processing — Isolated processing pipelines
🧪 Testing — Reproducible test environments

Docker vs Virtual Machines

  • Size: Docker containers are MB-sized; VMs are GB-sized
  • Startup: Containers start in seconds; VMs take minutes
  • Resources: Containers share host kernel; VMs include full OS
  • Portability: Container images run anywhere; VM images are hypervisor-specific
  • Management: Docker CLI is simpler than VM management tools
⚠️ Important: While Docker provides process isolation, it's not a complete security boundary like VMs. Use Docker in conjunction with other security practices for production workloads.

⚙️ Installation & Setup

Get started with Docker by installing Docker Desktop (Windows/macOS) or Docker Engine (Linux).

Windows Installation (Docker Desktop)

📌 Prerequisites: Windows 10/11 (64-bit) with WSL 2 enabled. Docker Desktop requires virtualization to be enabled in BIOS.

Option 1 — winget (Recommended)

# Install Docker Desktop
winget install -e --id Docker.DockerDesktop

# Enable WSL 2 if not already enabled
wsl --install

# Restart your computer after installation

Option 2 — Manual Download

1 Download Docker Desktop from docker.com
2 Run the installer and follow the setup wizard
3 Enable WSL 2 integration when prompted
4 Restart your computer
5 Launch Docker Desktop from Start menu

macOS Installation (Docker Desktop)

# Install Docker Desktop using Homebrew
brew install --cask docker

# Open Docker Desktop
open -a Docker

# Wait until Docker is running, then verify
docker --version
docker compose version

Alternative: Manual Download

Download Docker Desktop for Mac from docker.com (choose Apple Silicon or Intel chip)

Linux Installation (Docker Engine)

Ubuntu/Debian

# Remove old versions
sudo apt-get remove docker docker-engine docker.io containerd runc

# Set up repository
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings

# Add Docker's official GPG key
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Add repository
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker Engine
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# Start Dock service
sudo systemctl enable docker
sudo systemctl start docker

RHEL/CentOS/Fedora

# Remove old versions
sudo dnf remove docker docker-client docker-client-latest docker-common \
  docker-latest docker-latest-logrotate docker-logrotate docker-engine

# Set up repository
sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo \
  https://download.docker.com/linux/fedora/docker-ce.repo

# Install Docker Engine
sudo dnf install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# Start Docker service
sudo systemctl enable docker
sudo systemctl start docker

Working with Images

Pulling Images from Docker Hub

# Pull latest version
docker pull nginx

# Pull specific version
docker pull nginx:1.25-alpine
docker pull postgres:16
docker pull node:20-alpine

# Search for images
docker search redis

# View local images
docker images

# View image history
docker history nginx

# Remove image
docker rmi nginx:latest

# Remove all unused images
docker image prune -a

Building Custom Images

# Create Dockerfile
cat > Dockerfile << 'EOF'
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
EOF

# Build image
docker build -t myapp:1.0 .

# Build with build arguments
docker build --build-arg NODE_ENV=production -t myapp:prod .

# Build and tag multiple versions
docker build -t myapp:1.0 -t myapp:latest .

# View build history
docker history myapp:1.0

Multi-Stage Builds

# Dockerfile with multi-stage build
FROM node:20 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY --from=builder /app/dist ./dist
CMD ["node", "dist/server.js"]

Running Containers

Basic Container Operations

# Run container in foreground
docker run nginx

# Run in detached mode (background)
docker run -d nginx

# Run with custom name
docker run -d --name my-nginx nginx

# Run and remove after exit
docker run --rm ubuntu echo "Hello World"

# Run interactively
docker run -it ubuntu bash

# Run with port mapping
docker run -d -p 8080:80 nginx
# Host port 8080 -> Container port 80

# Run with environment variables
docker run -d -e DB_PASSWORD=secret postgres

# Run with volume mount
docker run -d -v /path/on/host:/path/in/container nginx

# Run with restart policy
docker run -d --restart unless-stopped nginx

Managing Containers

# List running containers
docker ps

# List all containers
docker ps -a

# Stop container
docker stop my-nginx

# Start stopped container
docker start my-nginx

# Restart container
docker restart my-nginx

# Pause container
docker pause my-nginx
docker unpause my-nginx

# Remove container
docker rm my-nginx

# Force remove running container
docker rm -f my-nginx

# Remove all stopped containers
docker container prune

Interacting with Containers

# View logs
docker logs my-nginx

# Follow logs (like tail -f)
docker logs -f my-nginx

# View last 100 lines
docker logs --tail 100 my-nginx

# Execute command in running container
docker exec my-nginx ls /usr/share/nginx/html

# Get interactive shell
docker exec -it my-nginx bash

# Copy files to/from container
docker cp file.txt my-nginx:/app/
docker cp my-nginx:/app/output.log ./

# View container details
docker inspect my-nginx

# View container stats (CPU, memory)
docker stats my-nginx

# View processes in container
docker top my-nginx

Dockerfile Best Practices

Well-Optimized Dockerfile

# Use specific base image version
FROM node:20-alpine AS base

# Set working directory
WORKDIR /app

# Copy only dependency files first (for caching)
COPY package*.json ./

# Install dependencies
RUN npm ci --only=production \
    && npm cache clean --force

# Copy application code
COPY . .

# Create non-root user
RUN addgroup -g 1001 -S nodejs \
    && adduser -S nodejs -u 1001 \
    && chown -R nodejs:nodejs /app

# Switch to non-root user
USER nodejs

# Expose port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node healthcheck.js

# Start application
CMD ["node", "server.js"]

Docker Compose

Basic docker-compose.yml

version: '3.8'

services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgres://postgres:password@db:5432/myapp
    depends_on:
      - db
    volumes:
      - ./src:/app/src
    restart: unless-stopped

  db:
    image: postgres:16-alpine
    environment:
      - POSTGRES_PASSWORD=password
      - POSTGRES_DB=myapp
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    restart: unless-stopped

volumes:
  postgres_data:

# Start services
docker compose up

docker compose up -d

# Build and start
docker compose up --build

docker compose stop

# Stop and remove containers
docker compose down

docker compose down -v

🚀 Advanced Topics

Docker Networking

# List networks
docker network ls

# Create custom network
docker network create my-network

# Run container in network
docker run -d --network my-network --name web nginx

# Connect container to network
docker network connect my-network my-container

# Inspect network
docker network inspect my-network

# Remove network
docker network rm my-network

Docker Volumes

# Create named volume
docker volume create my-data

# List volumes
docker volume ls

# Run container with volume
docker run -d -v my-data:/data nginx

# Inspect volume
docker volume inspect my-data

# Remove volume
docker volume rm my-data

# Remove all unused volumes
docker volume prune

# Backup volume
docker run --rm -v my-data:/data -v $(pwd):/backup \
  ubuntu tar czf /backup/backup.tar.gz /data

Docker Registry

# Login to Docker Hub
docker login

# Tag image for registry
docker tag myapp:latest username/myapp:latest

# Push to Docker Hub
docker push username/myapp:latest

# Pull from private registry
docker pull registry.company.com/myapp:latest

# Run local registry
docker run -d -p 5000:5000 --name registry registry:2

# Push to local registry
docker tag myapp:latest localhost:5000/myapp:latest
docker push localhost:5000/myapp:latest

Resource Limits

# Limit memory
docker run -d --memory="512m" nginx

# Limit CPU
docker run -d --cpus="1.5" nginx

# Limit both
docker run -d --memory="1g" --cpus="2" nginx

# Set memory swap limit
docker run -d --memory="512m" --memory-swap="1g" nginx

Docker System Management

# View disk usage
docker system df

# Remove unused data
docker system prune

# Remove everything including volumes
docker system prune -a --volumes

# View real-time events
docker events

# Export container
docker export my-container > container.tar

# Import image from tarball
docker import container.tar myimage:latest
