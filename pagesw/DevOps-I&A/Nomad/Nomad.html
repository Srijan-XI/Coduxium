🧭 Nomad

Nomad is a lightweight workload orchestrator for containers, virtual machines, and non-containerized applications.

Overview

Orchestration HashiCorp Multi-Workload Simple Ops

Nomad offers a straightforward scheduling model with minimal operational overhead, making it suitable for teams that need orchestration without Kubernetes complexity.

Core Concepts

  • Job — declarative workload specification
  • Task group — colocated tasks sharing resources
  • Allocation — scheduled job instance on a node
  • Datacenters/regions — topology-aware scheduling

Getting Started

# Dev mode server
nomad agent -dev

# Check status
nomad status
# Run a sample job
nomad job run example.nomad
nomad job status example

Common Use Cases

  • Container and non-container mixed orchestration
  • Batch scheduling and long-running services
  • Multi-region workload placement
  • Simplified orchestration for platform teams
