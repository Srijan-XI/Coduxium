📚 Introduction

What is GitHub?

GitHub is the world's leading platform for version control and collaboration, built on top of Git. It allows developers to host and review code, manage projects, and build software alongside millions of other developers. With over 100 million developers and 330 million repositories, GitHub has become the home of open source and enterprise software development.

Key Features

📦 Repositories — Host unlimited public and private repos
🔀 Pull Requests — Code review and collaboration
🐛 Issues — Track bugs, features, and tasks
⚙️ GitHub Actions — CI/CD automation workflows
📊 Projects — Kanban boards for project management
🔒 Security — Dependabot, code scanning, secret scanning
📚 Pages — Host static websites for free
🤝 Collaboration — Teams, organizations, discussions

Why Choose GitHub?

  • Industry Standard: Used by virtually every major tech company and millions of open-source projects.
  • Collaboration Tools: Pull requests, code review, and discussions make teamwork seamless.
  • CI/CD Integration: GitHub Actions provides unlimited CI/CD minutes for public repos.
  • Open Source Hub: Discover, contribute to, and learn from millions of open-source projects.
  • Professional Network: Your GitHub profile serves as your developer portfolio.
  • Ecosystem: Integrations with thousands of tools and services.

Use Cases

💻 Software Development — Host code and collaborate
🌐 Open Source — Contribute to or maintain projects
📝 Documentation — Version control for docs with GitHub Pages
🎓 Education — Learn, teach, and showcase projects
📦 Package Hosting — npm, Docker, Maven via GitHub Packages
⚙️ Automation — CI/CD, release automation, bot workflows

GitHub vs Alternatives

  • vs GitLab: Larger community, better for open source, simpler UI, free CI/CD for public repos
  • vs Bitbucket: Better third-party integrations, larger ecosystem, more active community
  • vs Azure DevOps: Easier to use, better for open source, more social features
⚠️ Important: Never commit secrets (API keys, passwords, tokens) to public repositories. Use GitHub Secrets for CI/CD and environment variables for applications.

⚙️ Installation & Setup

Get started with GitHub by creating an account and installing GitHub CLI for command-line access.

Create GitHub Account

1 Go to github.com
2 Click "Sign up" and enter your email
3 Create a password (strong password recommended)
4 Choose a username (this will be your profile URL)
5 Verify your email address
6 Complete the setup wizard

Account Types

  • Free: Unlimited public/private repos, Actions minutes (2,000/month), code spaces storage
  • Pro ($4/month): Advanced tools, insights, and support
  • Team ($4/user/month): Team access controls and management
  • Enterprise: Advanced security, compliance, and deployment

GitHub CLI (gh) Installation

The GitHub CLI brings GitHub to your terminal, allowing you to manage repositories, pull requests, issues, and more.

Windows Installation

# Using winget (recommended)
winget install -e --id GitHub.cli

# Using Chocolatey
choco install gh

# Using Scoop
scoop install gh

# Verify installation
gh --version

macOS Installation

# Using Homebrew (recommended)
brew install gh

# Verify installation
gh --version

Linux Installation

# Debian/Ubuntu
type -p curl >/dev/null || sudo apt install curl -y
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | \
  sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
sudo chmod go+r /usr/share/keyrings/githubcli-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] \
  https://cli.github.com/packages stable main" | \
  sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
sudo apt update
sudo apt install gh -y

# Fedora/RHEL
sudo dnf install gh

# Arch Linux
sudo pacman -S github-cli

# Verify installation
gh --version

Authentication

Authenticate GitHub CLI

# Start authentication flow
gh auth login

# Follow prompts:
# 1. GitHub.com or GitHub Enterprise Server? → GitHub.com
# 2. HTTPS or SSH? → HTTPS (recommended for beginners)
# 3. Authenticate Git with GitHub credentials? → Yes
# 4. How to authenticate? → Login with a web browser

# Verify authentication
gh auth status

# View current user
gh api user --jq '.login'

SSH Authentication (Alternative)

# Generate SSH key (if you haven't already)
ssh-keygen -t ed25519 -C "your.email@example.com"

# Start SSH agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Add SSH key to GitHub account
gh ssh-key add ~/.ssh/id_ed25519.pub -t "My Laptop"

# Or copy and add manually
cat ~/.ssh/id_ed25519.pub
# Go to GitHub → Settings → SSH and GPG keys → New SSH key

# Test connection
ssh -T git@github.com

Personal Access Tokens

1 Go to GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2 Click "Generate new token (classic)"
3 Give it a descriptive name
4 Set expiration (recommend 90 days)
5 Select scopes (repo, workflow, etc.)
6 Generate and copy token (save it securely!)
# Use token for HTTPS authentication
git clone https://USERNAME:TOKEN@github.com/user/repo.git

# Or configure git credential helper
git config --global credential.helper store
# Next time you push, enter token as password

📖 User Guide

Learn how to create repositories, collaborate with pull requests, manage issues, and automate workflows.

Working with Repositories

Create Repository (Web)

1 Click "+" in top-right → "New repository"
2 Choose owner (personal or organization)
3 Enter repository name
4 Add description (optional but recommended)
5 Choose Public or Private
6 Initialize with README, .gitignore, license (optional)
7 Click "Create repository"

Create Repository (CLI)

# Create new repo from current directory
gh repo create my-project --public --source=. --remote=origin --push

# Create empty repo
gh repo create username/my-project --public

# Create with description
gh repo create my-app --description "My awesome app" --public

# Create private repo
gh repo create my-secret-project --private

# Initialize from template
gh repo create my-project --template username/template-repo --public

Clone Repository

# Clone via HTTPS
gh repo clone username/repo

# Clone via SSH
git clone git@github.com:username/repo.git

# Clone to specific folder
gh repo clone username/repo my-folder

# Clone and immediately navigate
gh repo clone username/repo && cd repo

Repository Settings

# View repository information
gh repo view

# View repository in browser
gh repo view --web

# Archive repository
gh repo archive username/old-project

# Delete repository
gh repo delete username/old-project

# Set repository visibility
gh repo edit --visibility private

Branches & Pull Requests

Create Feature Branch

# Create and switch to new branch
git checkout -b feature/add-login

# Make changes and commit
git add .
git commit -m "feat: add login functionality"

# Push branch to GitHub
git push -u origin feature/add-login

Create Pull Request

# Create PR with auto-filled title and body from commits
gh pr create --fill

# Create PR with custom title and body
gh pr create --title "Add login feature" --body "Implements user authentication"

# Create PR targeting specific branch
gh pr create --base develop --head feature/add-login

# Create draft PR
gh pr create --draft --fill

# Create PR with labels and reviewers
gh pr create --fill --label enhancement --reviewer @username --assignee @me

Review Pull Requests

# List open PRs
gh pr list

# View PR details
gh pr view 123

# View PR in browser
gh pr view 123 --web

# Checkout PR locally
gh pr checkout 123

# Review PR
gh pr review 123 --approve
gh pr review 123 --request-changes --body "Please fix formatting"
gh pr review 123 --comment --body "Looks good!"

# Add comment to PR
gh pr comment 123 --body "Great work!"

Merge Pull Requests

# Merge PR (create merge commit)
gh pr merge 123

# Squash merge (recommended for feature branches)
gh pr merge 123 --squash

# Rebase merge
gh pr merge 123 --rebase

# Merge and delete branch
gh pr merge 123 --squash --delete-branch

# Auto-merge when checks pass
gh pr merge 123 --auto --squash

Managing Issues

Create Issues

# Create issue interactively
gh issue create

# Create with title and body
gh issue create --title "Bug: App crashes on startup" --body "Steps to reproduce..."

# Create with labels
gh issue create --title "Feature request" --label enhancement,help-wanted

# Create and assign
gh issue create --title "Fix tests" --assignee @me

# Create from template
gh issue create --template bug_report.md

Manage Issues

# List issues
gh issue list

# List open issues assigned to you
gh issue list --assignee @me

# List issues with specific label
gh issue list --label bug

# View issue details
gh issue view 42

# View issue in browser
gh issue view 42 --web

# Comment on issue
gh issue comment 42 --body "Working on this"

# Close issue
gh issue close 42

# Reopen issue
gh issue reopen 42

Link Issues to PRs

# In PR description or commit message, use keywords:
# "Fixes #42" or "Closes #42" or "Resolves #42"

# Create PR that closes issue
gh pr create --fill --body "Closes #42"

# Link existing PR to issue
gh pr edit 123 --add-label "fixes #42"

GitHub Actions (CI/CD)

Basic CI Workflow

# Create .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run linter
        run: npm run lint
      
      - name: Run tests
        run: npm test
      
      - name: Build
        run: npm run build

Advanced Workflow with Matrix

name: Test Matrix

on: [push, pull_request]

jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        node: [18, 20, 22]
    
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm ci
      - run: npm test

Deploy Workflow

name: Deploy to Production

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: production
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Build Docker image
        run: docker build -t myapp:${{ github.sha }} .
      
      - name: Push to registry
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin
          docker push myapp:${{ github.sha }}
      
      - name: Deploy to server
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.HOST }}
          username: ${{ secrets.USERNAME }}
          key: ${{ secrets.SSH_KEY }}
          script: |
            docker pull myapp:${{ github.sha }}
            docker stop myapp || true
            docker run -d --name myapp -p 80:3000 myapp:${{ github.sha }}

Scheduled Workflows (Cron Jobs)

name: Nightly Build

on:
  schedule:
    # Run at 2 AM UTC every day
    - cron: '0 2 * * *'
  workflow_dispatch:  # Allow manual trigger

jobs:
  nightly:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run nightly tasks
        run: |
          npm ci
          npm run build
          npm run test:e2e

Docker Build and Push to GHCR

name: Docker Build and Push

on:
  push:
    branches: [ main ]
    tags: [ 'v*.*.*' ]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3
      
      - name: Log in to Container Registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      
      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=ref,event=branch
            type=semver,pattern={{version}}
            type=sha
      
      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

Reusable Workflows

# .github/workflows/reusable-test.yml
name: Reusable Test Workflow

on:
  workflow_call:
    inputs:
      node-version:
        required: true
        type: string

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ inputs.node-version }}
      - run: npm ci
      - run: npm test

# Use in another workflow:
# .github/workflows/ci.yml
name: CI
on: [push]

jobs:
  call-test:
    uses: ./.github/workflows/reusable-test.yml
    with:
      node-version: '20'

Composite Actions

# .github/actions/setup-node-app/action.yml
name: 'Setup Node App'
description: 'Setup Node.js and install dependencies'

inputs:
  node-version:
    description: 'Node.js version'
    required: false
    default: '20'

runs:
  using: composite
  steps:
    - uses: actions/setup-node@v4
      with:
        node-version: ${{ inputs.node-version }}
        cache: 'npm'
    - run: npm ci
      shell: bash

# Use in workflow:
steps:
  - uses: actions/checkout@v4
  - uses: ./.github/actions/setup-node-app
    with:
      node-version: '20'

Release Automation

name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Build artifacts
        run: |
          npm ci
          npm run build
          zip -r release.zip dist/
      
      - name: Create Release
        uses: softprops/action-gh-release@v1
        with:
          files: release.zip
          generate_release_notes: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

GitHub Pages

# Deploy static site from gh-pages branch
# 1. Create gh-pages branch
git checkout --orphan gh-pages
git rm -rf .

# 2. Add your static files
echo "Hello World" > index.html
git add index.html
git commit -m "Initial GitHub Pages"
git push origin gh-pages

# 3. Enable in Settings → Pages → Source: gh-pages

# Automated deployment with Actions
name: Deploy to GitHub Pages
on:
  push:
    branches: [ main ]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npm run build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist

Branch Protection Rules

Enforce code quality and review requirements:

1 Go to Settings → Branches
2 Add branch protection rule for "main"
3 Enable "Require pull request reviews"
4 Enable "Require status checks to pass"
5 Enable "Require branches to be up to date"
6 Optionally require signed commits
