📚 Introduction

What is Git?

Git is a free and open-source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. Created by Linus Torvalds in 2005, Git has become the de facto standard for version control, used by millions of developers worldwide.

Key Features

🌿 Branching — Lightweight, fast branch creation and switching
🔄 Distributed — Every developer has full project history
⚡ Performance — Optimized for speed and efficiency
🔒 Data Integrity — Checksum everything with SHA-1
🎯 Staging Area — Fine-grained control over commits
📌 Tags — Mark release points and important commits
🔙 History Rewriting — Clean up commits before sharing
🤝 Collaboration — Merge, rebase, and resolve conflicts

Why Choose Git?

  • Industry Standard: Used by virtually every major tech company and open-source project.
  • Distributed: Work offline, commit locally, and sync when ready. No single point of failure.
  • Fast & Efficient: Operations are performed locally, making them lightning fast.
  • Powerful Branching: Create branches in seconds, switch instantly, and merge effortlessly.
  • Data Integrity: Cryptographic hashing ensures your code history is tamper-proof.
  • Open Source: Free forever with a vibrant community and extensive tooling ecosystem.

Use Cases

💻 Software Development — Track code changes and collaborate
📚 Documentation — Version control for docs and wikis
⚙️ Configuration Management — Track infrastructure as code
🎨 Design Files — Version graphics and creative assets
📊 Data Science — Version datasets and notebooks
✍️ Writing — Books, articles, research papers

Git Workflow Concepts

  • Working Directory: Your local files where you make changes
  • Staging Area (Index): Where you prepare commits
  • Repository (.git): Where Git stores project history
  • Remote Repository: Shared repository (GitHub, GitLab, Bitbucket)
  • Branch: Independent line of development
  • HEAD: Pointer to current branch/commit
⚠️ Important: Git stores snapshots, not differences. This makes operations like branching and merging extremely fast compared to older version control systems.

Initial Configuration

Set Your Identity

# Set your name (required)
git config --global user.name "Your Name"

# Set your email (required)
git config --global user.email "your.email@example.com"

# View configuration
git config --list

# View specific setting
git config user.name

Configure Default Editor

# VS Code
git config --global core.editor "code --wait"

# Vim
git config --global core.editor "vim"

# Nano
git config --global core.editor "nano"

# Sublime Text
git config --global core.editor "subl -n -w"

Set Default Branch Name

# Use 'main' instead of 'master'
git config --global init.defaultBranch main

Useful Aliases

# Short status
git config --global alias.st status

# Short log
git config --global alias.lg "log --oneline --graph --all"

# Undo last commit
git config --global alias.undo "reset HEAD~1"

# View aliases
git config --global --get-regexp alias

SSH Keys for GitHub/GitLab

Generate SSH Key

# Generate new SSH key
ssh-keygen -t ed25519 -C "your.email@example.com"

# For older systems without ed25519
ssh-keygen -t rsa -b 4096 -C "your.email@example.com"

# Press Enter to accept default location (~/.ssh/id_ed25519)
# Enter passphrase (recommended but optional)

Add SSH Key to SSH Agent

# Start SSH agent
eval "$(ssh-agent -s)"

# Add private key
ssh-add ~/.ssh/id_ed25519

Add Public Key to GitHub

# Copy public key to clipboard

# macOS
pbcopy < ~/.ssh/id_ed25519.pub

# Linux (with xclip)
xclip -selection clipboard < ~/.ssh/id_ed25519.pub

# Windows (Git Bash)
cat ~/.ssh/id_ed25519.pub | clip

# Or just display it
cat ~/.ssh/id_ed25519.pub

Then go to GitHub → Settings → SSH and GPG keys → New SSH key → Paste and save

Test SSH Connection

# Test GitHub connection
ssh -T git@github.com

# Test GitLab connection
ssh -T git@gitlab.com

Verify Setup

# Check Git version
git --version

# View all configuration
git config --list --show-origin

# Check user settings
git config user.name
git config user.email

# Test creating a repository
mkdir test-repo
cd test-repo
git init

📖 User Guide

Master essential Git commands and workflows for effective version control.

Getting Started

Initialize a Repository

# Create new repository
git init

# Create and initialize in one step
git init my-project
cd my-project

Clone Existing Repository

# Clone via HTTPS
git clone https://github.com/username/repo.git

# Clone via SSH
git clone git@github.com:username/repo.git

# Clone to specific folder
git clone https://github.com/username/repo.git my-folder

# Clone specific branch
git clone -b develop https://github.com/username/repo.git

Basic Workflow

Check Status

# View working tree status
git status

# Short status
git status -s

Stage Changes

# Stage specific file
git add filename.txt

# Stage all changes in current directory
git add .

# Stage all changes in repository
git add -A

# Stage interactively (choose hunks)
git add -p filename.txt

# Stage all modified files (not new files)
git add -u

Commit Changes

# Commit with message
git commit -m "Add login feature"

# Commit with multi-line message
git commit -m "Add login feature" -m "- Email validation\n- Password hashing\n- Session management"

# Stage and commit in one step
git commit -am "Fix bug in user auth"

# Amend last commit
git commit --amend -m "Corrected commit message"

# Amend without changing message
git commit --amend --no-edit

View History

# View commit history
git log

# One line per commit
git log --oneline

# Graph view
git log --oneline --graph --all

# Last N commits
git log -5

# Show changes in commits
git log -p

# Pretty format
git log --pretty=format:"%h - %an, %ar : %s"

# Filter by author
git log --author="John"

# Filter by date
git log --since="2 weeks ago"
git log --after="2025-01-01" --before="2025-12-31"

Branching & Merging

Branch Operations

# List branches
git branch

# List all branches (including remote)
git branch -a

# Create new branch
git branch feature/login

# Switch to branch
git checkout feature/login

# Create and switch in one step
git checkout -b feature/login

# Modern way (Git 2.23+)
git switch feature/login
git switch -c feature/new-feature

# Rename branch
git branch -m old-name new-name

# Delete branch
git branch -d feature/login

# Force delete unmerged branch
git branch -D feature/login

Merging

# Merge branch into current
git checkout main
git merge feature/login

# Merge with commit message
git merge feature/login -m "Merge login feature"

# Merge without fast-forward
git merge --no-ff feature/login

# Abort merge
git merge --abort

# Continue after resolving conflicts
git merge --continue

Rebase

# Rebase current branch onto main
git checkout feature/login
git rebase main

# Interactive rebase (last 5 commits)
git rebase -i HEAD~5

# Continue after resolving conflicts
git rebase--continue

# Skip commit during rebase
git rebase --skip

# Abort rebase
git rebase --abort

Remote Repositories

Managing Remotes

# View remotes
git remote -v

# Add remote
git remote add origin https://github.com/username/repo.git

# Change remote URL
git remote set-url origin git@github.com:username/repo.git

# Remove remote
git remote remove origin

# Rename remote
git remote rename origin upstream

Fetching & Pulling

# Fetch from remote (doesn't merge)
git fetch origin

# Fetch all remotes
git fetch --all

# Pull (fetch + merge)
git pull

# Pull with rebase
git pull --rebase

# Pull specific branch
git pull origin main

Pushing

# Push to remote
git push origin main

# Push and set upstream
git push -u origin feature/login

# Push all branches
git push --all

# Push tags
git push --tags

# Force push (dangerous!)
git push --force

# Safer force push
git push --force-with-lease

# Delete remote branch
git push origin --delete feature/old-branch
-- Undoing Changes -->

Undoing Changes

Discard Changes

# Discard changes in working directory
git restore filename.txt

# Discard all changes
git restore .

# Unstage file (keep changes)
git restore --staged filename.txt

# Old way (still works)
git checkout -- filename.txt

Reset Commits

# Undo last commit (keep changes staged)
git reset --soft HEAD~1

# Undo last commit (keep changes unstaged)
git reset HEAD~1
git reset --mixed HEAD~1

# Undo last commit (discard changes)
git reset --hard HEAD~1

# Reset to specific commit
git reset --hard abc123

# Reset specific file
git reset HEAD filename.txt

Revert Commits

# Create new commit that undoes changes
git revert HEAD

# Revert specific commit
git revert abc123

# Revert without committing
git revert --no-commit HEAD

Stashing Changes

# Stash changes
git stash

# Stash with message
git stash save "Work in progress on login"

# List stashes
git stash list

# Apply latest stash
git stash apply

# Apply specific stash
git stash apply stash@{2}

# Apply and remove stash
git stash pop

# Show stash contents
git stash show -p

# Delete stash
git stash drop stash@{0}

# Delete all stashes
git stash clear

🚀 Advanced Topics

Tags

# Create lightweight tag
git tag v1.0.0

# Create annotated tag
git tag -a v1.0.0 -m "Release version 1.0.0"

# Tag specific commit
git tag -a v1.0.0 abc123

# List tags
git tag

# Show tag info
git show v1.0.0

# Push tags
git push origin v1.0.0
git push origin --tags

# Delete tag
git tag -d v1.0.0
git push origin --delete v1.0.0

Cherry-Pick

# Apply specific commit to current branch
git cherry-pick abc123

# Cherry-pick multiple commits
git cherry-pick abc123 def456

# Cherry-pick without committing
git cherry-pick --no-commit abc123

Bisect (Find Bugs)

# Start bisect
git bisect start

# Mark current as bad
git bisect bad

# Mark known good commit
git bisect good abc123

# Git will checkout middle commit
# Test and mark as good or bad
git bisect good  # or
git bisect bad

# Repeat until bug is found

# Reset after done
git bisect reset

Submodules

# Add submodule
git submodule add https://github.com/user/repo.git path/to/submodule

# Initialize submodules after clone
git submodule init
git submodule update

# Or in one step
git submodule update --init --recursive

# Update submodules
git submodule update --remote

# Remove submodule
git submodule deinit path/to/submodule
git rm path/to/submodule

Hooks

# Hooks are scripts in .git/hooks/
# Example: pre-commit hook

# Create pre-commit hook
cat > .git/hooks/pre-commit << 'EOF'
#!/bin/sh
# Run tests before commit
npm test
EOF

# Make executable
chmod +x .git/hooks/pre-commit

# Common hooks:
# - pre-commit: Before commit
# - pre-push: Before push
# - post-merge: After merge
# - commit-msg: Validate commit message

.gitignore

# Create .gitignore file
cat > .gitignore << 'EOF'

# Build outputs
