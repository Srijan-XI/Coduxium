🧵 Zipkin

Zipkin is a distributed tracing system used to collect and visualize request latency across microservices.

Overview

ObservabilityTracingMicroservicesLatency

Zipkin helps teams understand service dependencies and pinpoint bottlenecks by tracing end-to-end request paths.

Common Use Cases

  • Root-cause analysis for latency regressions
  • Cross-service dependency mapping
  • Performance troubleshooting in distributed systems
  • Tracing verification during release rollouts

Getting Started

# Run Zipkin locally
docker run -d -p 9411:9411 openzipkin/zipkin

# UI
http://localhost:9411
