🏗️ OpenTofu

OpenTofu is an open-source infrastructure-as-code tool for defining, planning, and provisioning cloud resources declaratively.

Overview

IaC Declarative State Management Cloud Provisioning

OpenTofu supports predictable infrastructure changes through plan/apply workflows, reusable modules, and policy-friendly automation.

Core Concepts

  • Configuration files — resource definitions in HCL
  • State — tracked resource mapping and drift context
  • Plan — preview proposed infrastructure changes
  • Modules — reusable infrastructure building blocks

Getting Started

# Initialize working directory
tofu init

# Validate and preview changes
tofu validate
tofu plan
# Apply safely
tofu apply

# Destroy when needed
tofu destroy

Best Practices

  • Use remote state backends with locking enabled.
  • Split environments and stacks into clear boundaries.
  • Review plan outputs in CI before apply.
  • Pin provider versions and module references.
