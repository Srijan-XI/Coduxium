🏗️ Terraform

Infrastructure as Code tool for building, changing, and versioning infrastructure safely and efficiently.

Overview

Infrastructure as Code DevOps Cloud HashiCorp

Terraform is an open-source infrastructure as code software tool created by HashiCorp. Users define and provide data center infrastructure using a declarative configuration language known as HashiCorp Configuration Language (HCL).

Key Features

  • Multi-Cloud Support — AWS, Azure, GCP, and 1000+ providers
  • Declarative Configuration — Define what you want, not how to get it
  • State Management — Track infrastructure state
  • Plan Before Apply — Preview changes before execution
  • Resource Graph — Build dependency graphs
  • Modules — Reusable infrastructure components
  • Workspaces — Manage multiple environments
  • Remote State — Collaboration with shared state

Common Use Cases

  • Multi-cloud infrastructure deployment
  • Infrastructure automation
  • Self-service infrastructure
  • Kubernetes cluster provisioning
  • Network infrastructure management
  • Database and storage provisioning
  • GitOps workflows
  • Disaster recovery automation

Getting Started

Install Terraform:

# Download from terraform.io or use package manager
# Linux
wget https://releases.hashicorp.com/terraform/1.6.0/terraform_1.6.0_linux_amd64.zip
unzip terraform_1.6.0_linux_amd64.zip
sudo mv terraform /usr/local/bin/

# Verify installation
terraform version

Basic Terraform configuration:

# main.tf
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  
  tags = {
    Name = "MyWebServer"
  }
}

output "instance_ip" {
  value = aws_instance.web.public_ip
}

Terraform workflow:

# Initialize
terraform init

# Plan changes
terraform plan

# Apply changes
terraform apply

# Destroy infrastructure
terraform destroy

Best Practices

  • Use remote state for team collaboration
  • Organize code with modules
  • Use variables for reusability
  • Implement state locking
  • Use workspaces for environments
  • Version control your Terraform code
  • Use terraform fmt for consistent formatting
  • Implement proper IAM permissions
