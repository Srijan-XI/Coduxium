📚 Introduction

What is Jenkins?

Jenkins is a self-contained, open-source automation server that can be used to automate all sorts of tasks related to building, testing, and delivering or deploying software. Written in Java and with over 1,800 plugins, Jenkins has become the de facto standard for CI/CD automation.

Key Features

🔄 Continuous Integration — Automated build and test on every commit
🚀 Continuous Delivery — Automated deployment pipelines
📜 Pipeline as Code — Define builds in Jenkinsfile
🔌 1,800+ Plugins — Extend functionality infinitely
🖥️ Distributed Builds — Master-agent architecture
🔒 Security — Role-based access control
🌐 Open Source — Free forever with active community
☁️ Cloud Integration — AWS, Azure, GCP, Kubernetes

Why Choose Jenkins?

  • Industry Standard: Most widely adopted CI/CD tool with proven reliability at scale.
  • Extensive Plugin Ecosystem: Integrate with virtually any tool in your software development lifecycle.
  • Flexible: Supports both declarative and scripted pipelines for any workflow.
  • Distributed Architecture: Scale horizontally with agent nodes for parallel builds.
  • Open Source: No licensing costs, ever. Community-driven development.
  • Active Community: Vast knowledge base, tutorials, and community support.

Use Cases

⚙️ Build Automation — Compile code on every commit
🧪 Automated Testing — Run unit, integration, E2E tests
🚀 Deployment — Deploy to staging/production
📊 Code Quality — SonarQube, linting, security scans
🐳 Container Builds — Build and push Docker images
📦 Artifact Management — Publish to Nexus, Artifactory

Jenkins Architecture

  • Controller (Master): Schedules builds, monitors agents, stores configuration
  • Agents (Slaves): Execute build jobs on different machines/containers
  • Executors: Slots on agents that run individual builds
  • Workspace: Directory where build executes
  • Pipeline: Automated process defined in Jenkinsfile
  • Plugins: Extend Jenkins functionality
⚠️ Important: Jenkins master should not run build jobs directly in production. Use dedicated agent nodes to ensure controller stability and security.

⚙️ Installation & Setup

Get started with Jenkins using Docker, package managers, or WAR file deployment.

Docker Installation (Recommended)

Run Jenkins with Docker

# Create volume for Jenkins data
docker volume create jenkins_home

# Run Jenkins LTS
docker run -d \
  --name jenkins \
  -p 8080:8080 \
 -p 50000:50000 \
  -v jenkins_home:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkins/jenkins:lts-jdk17

# View logs
docker logs -f jenkins

# Get initial admin password
docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword

Docker Compose

version: '3.8'

services:
  jenkins:
    image: jenkins/jenkins:lts-jdk17
    container_name: jenkins
    privileged: true
    user: root
    ports:
      - "8080:8080"
      - "50000:50000"
    volumes:
      - jenkins_home:/var/jenkins_home
      - /var/run/docker.sock:/var/run/docker.sock
    environment:
      - JAVA_OPTS=-Djenkins.install.runSetupWizard=false
    restart: unless-stopped

volumes:
  jenkins_home:

Jenkinsfile (Declarative Pipeline)

Basic Jenkinsfile

pipeline {
    agent any
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Build') {
            steps {
                sh 'npm install'
                sh 'npm run build'
            }
        }
        
        stage('Test') {
            steps {
                sh 'npm test'
            }
        }
        
        stage('Deploy') {
            when {
                branch 'main'
            }
            steps {
                sh 'npm run deploy'
            }
        }
    }
    
    post {
        always {
            cleanWs()
        }
        success {
            echo 'Pipeline succeeded!'
        }
        failure {
            echo 'Pipeline failed!'
        }
    }
}

Advanced Jenkinsfile

pipeline {
    agent any
    
    environment {
        NODE_VERSION = '20'
        DOCKER_REGISTRY = 'docker.io'
        IMAGE_NAME = 'myapp'
        IMAGE_TAG = "${env.BUILD_NUMBER}"
    }
    
    parameters {
        choice(name: 'ENVIRONMENT', choices: ['dev', 'staging', 'prod'], description: 'Deployment environment')
        booleanParam(name: 'RUN_TESTS', defaultValue: true, description: 'Run tests?')
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Install Dependencies') {
            steps {
                sh 'npm ci'
            }
        }
        
        stage('Lint') {
            steps {
                sh 'npm run lint'
            }
        }
        
        stage('Test') {
            when {
                expression { params.RUN_TESTS == true }
            }
            steps {
                sh 'npm test -- --ci --coverage'
            }
            post {
                always {
                    junit 'junit.xml'
                    publishHTML([
                        reportDir: 'coverage',
                        reportFiles: 'index.html',
                        reportName: 'Coverage Report'
                    ])
                }
            }
        }
        
        stage('Build Docker Image') {
            steps {
                script {
                    docker.build("${DOCKER_REGISTRY}/${IMAGE_NAME}:${IMAGE_TAG}")
                }
            }
        }
        
        stage('Push Docker Image') {
            when {
                branch 'main'
            }
            steps {
                script {
                    docker.withRegistry('', 'docker-hub-credentials') {
                        docker.image("${DOCKER_REGISTRY}/${IMAGE_NAME}:${IMAGE_TAG}").push()
                        docker.image("${DOCKER_REGISTRY}/${IMAGE_NAME}:${IMAGE_TAG}").push('latest')
                    }
                }
            }
        }
        
        stage('Deploy') {
            when {
                branch 'main'
            }
            steps {
                sh """
                    kubectl set image deployment/myapp \
                        myapp=${DOCKER_REGISTRY}/${IMAGE_NAME}:${IMAGE_TAG} \
                        -n ${params.ENVIRONMENT}
                """
            }
        }
    }
    
    post {
        always {
            cleanWs()
        }
        success {
            slackSend(color: 'good', message: "Build #${env.BUILD_NUMBER} succeeded!")
        }
        failure {
            slackSend(color: 'danger', message: "Build #${env.BUILD_NUMBER} failed!")
        }
    }
}

Managing Credentials

Add Credentials

1 Go to "Manage Jenkins" → "Credentials"
2 Click on "(global)" domain
3 Click "Add Credentials"
4 Choose type (Username/Password, SSH, Secret text, etc.)
5 Enter credentials and ID
6 Save

Use Credentials in Pipeline

pipeline {
    agent any
    
    environment {
        // Bind credentials as environment variables
        DOCKER_CREDS = credentials('docker-hub-credentials')
    }
    
    stages {
        stage('Use Credentials') {
            steps {
                // Username/Password
                withCredentials([usernamePassword(
                    credentialsId: 'docker-hub-credentials',
                    usernameVariable: 'DOCKER_USER',
                    passwordVariable: 'DOCKER_PASS'
                )]) {
                    sh 'docker login -u $DOCKER_USER -p $DOCKER_PASS'
                }
                
                // Secret text
                withCredentials([string(
                    credentialsId: 'api-token',
                    variable: 'API_TOKEN'
                )]) {
                    sh 'curl -H "Authorization: Bearer $API_TOKEN" api.example.com'
                }
                
                // SSH key
                sshagent(['ssh-credentials-id']) {
                    sh 'ssh user@server "deploy.sh"'
                }
            }
        }
    }
}

Configuring Agents

Add Agent Node

1 Go to "Manage Jenkins" → "Nodes"
2 Click "New Node"
3 Enter name and choose "Permanent Agent"
4 Configure remote root directory, labels, launch method
5 Save and launch agent

Use Specific Agent

pipeline {
    agent {
        label 'docker'  // Run on agent with 'docker' label
    }
    
    stages {
        stage('Build') {
            agent {
                docker {
                    image 'node:20-alpine'
                    args '-v $HOME/.npm:/.npm'
                }
            }
            steps {
                sh 'npm install'
                sh 'npm run build'
            }
        }
    }
}

Parallel Execution

stages {
    stage('Parallel Tests') {
        parallel {
            stage('Unit Tests') {
                steps {
                    sh 'npm run test:unit'
                }
            }
            stage('Integration Tests') {
                steps {
                    sh 'npm run test:integration'
                }
            }
            stage('E2E Tests') {
                steps {
                    sh 'npm run test:e2e'
                }
            }
        }
    }
}

Shared Libraries

// vars/buildAndPush.groovy
def call(String imageName, String tag) {
    sh "docker build -t ${imageName}:${tag} ."
    sh "docker push ${imageName}:${tag}"
}

// Jenkinsfile
@Library('my-shared-library') _

pipeline {
    agent any
    stages {
        stage('Build and Push') {
            steps {
                buildAndPush('myapp', env.BUILD_NUMBER)
            }
        }
    }
}
