🧩 Kustomize

Kustomize customizes Kubernetes manifests using overlays and patches without introducing a templating language.

Overview

Kubernetes Config Management Overlays GitOps Friendly

Kustomize helps teams keep a clean base manifest set while layering environment-specific changes for dev, staging, and production.

Core Concepts

  • Base — shared manifests for all environments
  • Overlay — environment-specific customizations
  • Patches — targeted resource modifications
  • Transformers — labels, namespaces, image tags

Getting Started

# Preview output
kubectl kustomize ./overlays/dev

# Apply overlay
kubectl apply -k ./overlays/dev
# kustomization.yaml (example)
resources:
  - ../../base
namespace: myapp-dev
images:
  - name: myapp
    newTag: 1.2.3

Best Practices

  • Keep base manifests minimal and reusable.
  • Model overlays by environment and team boundary.
  • Use strategic merge patches for small, auditable changes.
  • Validate rendered output in CI before apply.
