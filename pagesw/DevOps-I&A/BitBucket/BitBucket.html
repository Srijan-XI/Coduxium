📚 Introduction

What is Bitbucket?

Bitbucket is a Git-based code hosting and collaboration platform by Atlassian. It provides unlimited private repositories, built-in CI/CD with Bitbucket Pipelines, and tight integration with Jira and Confluence for complete project management. Bitbucket is particularly popular among enterprise teams already using Atlassian products.

Key Features

📦 Unlimited Private Repos — Free for teams up to 5 users
🔄 Bitbucket Pipelines — Built-in CI/CD with YAML configuration
🔀 Pull Requests — Code review with inline comments
🎯 Jira Integration — Link commits, branches, and PRs to Jira issues
📝 Confluence Integration — Documentation alongside code
🔒 Branch Permissions — Fine-grained access control
📊 Insights — Code review metrics and analytics
🐳 Docker Support — Build with Docker in Pipelines

Why Choose Bitbucket?

  • Atlassian Ecosystem: Seamless integration with Jira, Confluence, and Trello.
  • Unlimited Private Repos: Free for small teams (up to 5 users).
  • Built-in CI/CD: Bitbucket Pipelines included with every repository.
  • Enterprise Features: IP whitelisting, merge checks, deployment tracking.
  • Monorepo Support: Good for large codebases and monolithic repositories.
  • Flexible Pricing: Pay for what you use with pipeline minutes.

Use Cases

💼 Enterprise Teams — Atlassian stack users
🔒 Private Projects — Unlimited private repositories
🔄 CI/CD Workflows — Integrated pipeline automation
🎯 Agile Development — Jira issue tracking integration

🛠️ Installation & Setup

Create Bitbucket Account

1 Go to bitbucket.org
2 Click "Get started for free"
3 Sign up with email or continue with Google/Microsoft
4 Create a workspace (team name)
5 Verify your email address

Pricing Tiers

  • Free: Up to 5 users, unlimited private repos, 50 build minutes/month
  • Standard ($3/user/month): Up to 100 users, 2,500 build minutes/month
  • Premium ($6/user/month): Unlimited users, deployment permissions, IP whitelisting

SSH Key Setup

# Generate SSH key
ssh-keygen -t ed25519 -C "your_email@example.com"

# Start SSH agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Copy public key
cat ~/.ssh/id_ed25519.pub

# Add to Bitbucket:
# Personal settings → SSH keys → Add key

# Test connection
ssh -T git@bitbucket.org

App Passwords (HTTPS)

# For HTTPS authentication, create app password:
# Personal settings → App passwords → Create app password
# Give it a label and select permissions

# Use when cloning/pushing:
git clone https://YOUR_USERNAME@bitbucket.org/workspace/repo.git
# Password: [paste app password]

# Save credentials
git config --global credential.helper store

Git Configuration

git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"

# Set default branch name
git config --global init.defaultBranch main

📖 User Guide

Creating Repositories

# Create repository via web UI:
# Workspace → Repositories → Create repository

# Clone repository
git clone git@bitbucket.org:workspace/repo-name.git
cd repo-name

# Create files and push
echo "# My Project" > README.md
git add README.md
git commit -m "Initial commit"
git push -u origin main

Branches and Pull Requests

# Create feature branch
git checkout -b feature/new-feature

# Make changes and commit
git add .
git commit -m "feat: add new feature"
git push -u origin feature/new-feature

# Create pull request:
# Repository → Pull requests → Create pull request
# Select source (feature/new-feature) and destination (main)
# Add reviewers and description

# Merge pull request:
# Review changes → Approve → Merge

Jira Integration

# Link commits to Jira issues by including issue key:
git commit -m "PROJ-123 Fix login bug"

# Create branch from Jira:
git checkout -b feature/PROJ-123-add-feature

# View linked commits in Jira issue
# Automatically transition issues with smart commits:
git commit -m "PROJ-123 #done Fixed login bug"
# Possible actions: #comment, #time, #done, #in-progress

Code Review

# Request reviewers on pull request
# Add inline comments on specific lines
# Use tasks to track action items

# Require approvals before merge:
# Repository settings → Branch permissions → Add rule
# Require X approvals before merge

# Default reviewers:
# Repository settings → Default reviewers
# Add users who should review all PRs

🔄 Bitbucket Pipelines

Basic Pipeline Configuration

Create bitbucket-pipelines.yml in repository root:

# bitbucket-pipelines.yml
image: node:20

pipelines:
  default:
    - step:
        name: Build and Test
        caches:
          - node
        script:
          - npm ci
          - npm run lint
          - npm test
          - npm run build
        artifacts:
          - dist/**

Branch-Specific Pipelines

image: node:20

pipelines:
  branches:
    main:
      - step:
          name: Test
          script:
            - npm ci
            - npm test
      - step:
          name: Build
          script:
            - npm run build
      - step:
          name: Deploy to Production
          deployment: production
          script:
            - ./deploy.sh production
    
    develop:
      - step:
          name: Deploy to Staging
          deployment: staging
          script:
            - ./deploy.sh staging
  
  pull-requests:
    '**':
      - step:
          name: PR Validation
          script:
            - npm ci
            - npm run lint
            - npm test

Docker Build Pipeline

image: atlassian/default-image:latest

pipelines:
  default:
    - step:
        name: Build Docker Image
        services:
          - docker
        script:
          - export IMAGE_NAME="myapp:${BITBUCKET_COMMIT}"
          - docker build -t $IMAGE_NAME .
          - docker login -u $DOCKER_USERNAME -p $DOCKER_PASSWORD
          - docker tag $IMAGE_NAME $DOCKER_USERNAME/myapp:latest
          - docker push $DOCKER_USERNAME/myapp:latest
          - docker push $IMAGE_NAME

definitions:
  services:
    docker:
      memory: 2048

Multi-Stage Pipeline

image: node:20

pipelines:
  default:
    - parallel:
        - step:
            name: Lint
            script:
              - npm ci
              - npm run lint
        - step:
            name: Test
            script:
              - npm ci
              - npm test
        - step:
            name: Security Scan
            script:
              - npm audit
    - step:
        name: Build
        script:
          - npm run build
        artifacts:
          - dist/**
    - step:
        name: Deploy
        deployment: production
        trigger: manual
        script:
          - ./deploy.sh

Deployment Environments

image: node:20

pipelines:
  branches:
    main:
      - step:
          name: Build
          script:
            - npm ci
            - npm run build
          artifacts:
            - dist/**
      - step:
          name: Deploy to Staging
          deployment: staging
          script:
            - pipe: atlassian/aws-s3-deploy:1.1.0
              variables:
                AWS_ACCESS_KEY_ID: $AWS_ACCESS_KEY_ID
                AWS_SECRET_ACCESS_KEY: $AWS_SECRET_ACCESS_KEY
                S3_BUCKET: 'staging-bucket'
                LOCAL_PATH: 'dist'
      - step:
          name: Deploy to Production
          deployment: production
          trigger: manual  # Require manual approval
          script:
            - pipe: atlassian/aws-s3-deploy:1.1.0
              variables:
                AWS_ACCESS_KEY_ID: $AWS_ACCESS_KEY_ID
                AWS_SECRET_ACCESS_KEY: $AWS_SECRET_ACCESS_KEY
                S3_BUCKET: 'production-bucket'
                LOCAL_PATH: 'dist'

Using Pipes (Pre-built Actions)

image: node:20

pipelines:
  default:
    - step:
        name: Deploy to AWS
        script:
          # AWS S3 Deploy Pipe
          - pipe: atlassian/aws-s3-deploy:1.1.0
            variables:
              AWS_ACCESS_KEY_ID: $AWS_KEY
              AWS_SECRET_ACCESS_KEY: $AWS_SECRET
              S3_BUCKET: 'my-bucket'
              LOCAL_PATH: 'dist'
          
          # Slack Notification Pipe
          - pipe: atlassian/slack-notify:2.0.0
            variables:
              WEBHOOK_URL: $SLACK_WEBHOOK
              MESSAGE: 'Deployment completed!'

🚀 Advanced Topics

Branch Permissions

# Repository settings → Branch permissions → Add rule

# Example: Protect main branch
Branch: main
Write access: Nobody (except merge)
Merge via pull request: Enabled
Require approvals: 2
Require passing builds: Enabled
Require tasks to be resolved: Enabled

Merge Strategies

# Repository settings → Merge strategies

# Available strategies:
# 1. Merge commit - Preserves complete history
# 2. Squash - Combines all commits into one
# 3. Fast-forward - Linear history (no merge commits)

# Set default merge strategy or allow reviewers to choose

Webhooks

# Repository settings → Webhooks → Add webhook

# Trigger events:
# - Repository push
# - Pull request created/updated/merged
# - Pull request comment created
# - Build status changed

# Example: Trigger Jenkins build
URL: https://jenkins.example.com/bitbucket-hook/
Events: Repository push

Downloads (Release Files)

# Upload release files to Downloads section
# Repository → Downloads → Add files

# Access download URL:
https://bitbucket.org/workspace/repo/downloads/

# Automate in pipeline:
pipelines:
  tags:
    'v*':
      - step:
          name: Create Release
          script:
            - npm run build
            - zip -r release-${BITBUCKET_TAG}.zip dist/
            - curl -X POST -u "${BB_AUTH_STRING}" \
              "https://api.bitbucket.org/2.0/repositories/${BITBUCKET_REPO_FULL_NAME}/downloads" \
              --form files=@"release-${BITBUCKET_TAG}.zip"

Issue Tracker

# Enable in Repository settings → Issue tracker

# Create issues directly in Bitbucket

Wiki

# Push changes to update wiki

Repository Access Tokens

# Repository settings → Access tokens → Create token
# Use for read-only or write access without user credentials

# Clone with access token:
git clone https://x-token-auth:${ACCESS_TOKEN}@bitbucket.org/workspace/repo.git

# Use in CI/CD:
pipelines:
  default:
    - step:
        script:
          - git clone https://x-token-auth:${REPO_TOKEN}@bitbucket.org/workspace/other-repo.git
