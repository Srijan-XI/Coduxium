📱 iOS

Apple's mobile operating system powering iPhone, iPad (as iPadOS), and iPod touch with seamless ecosystem integration.

Overview

MOBILE TOUCH SECURE ECOSYSTEM

iOS is Apple's mobile operating system, first introduced in 2007 with the original iPhone. Built on a Unix-like foundation similar to macOS, iOS prioritizes security, privacy, performance, and seamless integration across the Apple ecosystem.

Major iOS Versions

  • iOS 1.0 (2007) — Original iPhone OS, revolutionary multi-touch interface
  • iOS 2.0 (2008) — App Store launched, third-party apps
  • iOS 3.0 (2009) — Copy/paste, MMS, push notifications
  • iOS 4.0 (2010) — Multitasking, folders, FaceTime
  • iOS 5.0 (2011) — iCloud, Notification Center, iMessage
  • iOS 6.0 (2012) — Apple Maps (controversial), Passbook
  • iOS 7.0 (2013) — Complete redesign, flat UI, Control Center
  • iOS 8.0 (2014) — Continuity, extensibility, widgets
  • iOS 9.0 (2015) — iPad multitasking, Low Power Mode
  • iOS 10.0 (2016) — Redesigned Messages, HomeKit
  • iOS 11.0 (2017) — Files app, ARKit, drag & drop on iPad
  • iOS 12.0 (2018) — Screen Time, Performance focus
  • iOS 13.0 (2019) — Dark Mode, Sign in with Apple
  • iPadOS 13.1 (2019) — iPad gets separate OS
  • iOS 14.0 (2020) — Widgets on home screen, App Library, App Clips
  • iOS 15.0 (2021) — Focus modes, Live Text, SharePlay
  • iOS 16.0 (2022) — Lock Screen customization, Live Activities
  • iOS 17.0 (2023) — Contact Posters, StandBy mode, NameDrop
  • iOS 18.0 (2024) — Apple Intelligence, RCS messaging, customization

Key Features

  • Touch Interface: Multi-touch gestures, haptic feedback
  • Face ID/Touch ID: Biometric authentication
  • App Store: Curated app marketplace with 1.8M+ apps
  • iMessage: Rich messaging with end-to-end encryption
  • FaceTime: Video and audio calling
  • Siri: Intelligent voice assistant
  • Apple Pay: Contactless payments
  • Health: Comprehensive health and fitness tracking
  • Privacy: App Tracking Transparency, Privacy Labels
  • Continuity: Handoff, Universal Clipboard, AirDrop
  • ARKit: Augmented reality framework
  • Focus Modes: Customizable notification filters

iOS Development with Swift

import UIKit

// Simple View Controller
class ViewController: UIViewController {
    
    override func viewDidLoad() {
        super.viewDidLoad()
        
        // Create a label
        let label = UILabel()
        label.text = "Hello, iOS!"
        label.textAlignment = .center
        label.frame = CGRect(x: 50, y: 200, width: 300, height: 50)
        view.addSubview(label)
        
        // Create a button
        let button = UIButton(type: .system)
        button.setTitle("Tap Me", for: .normal)
        button.frame = CGRect(x: 100, y: 300, width: 200, height: 50)
        button.addTarget(self, action: #selector(buttonTapped), for: .touchUpInside)
        view.addSubview(button)
    }
    
    @objc func buttonTapped() {
        print("Button was tapped!")
        let alert = UIAlertController(
            title: "Alert",
            message: "Button tapped!",
            preferredStyle: .alert
        )
        alert.addAction(UIAlertAction(title: "OK", style: .default))
        present(alert, animated: true)
    }
}

// SwiftUI App Lifecycle
@main
struct MyApp: App {
    @Environment(\.scenePhase) var scenePhase
    
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        .onChange(of: scenePhase) { oldPhase, newPhase in

Push Notifications

import UserNotifications

// Request permission
UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .sound, .badge]) { granted, error in
    if granted {
        DispatchQueue.main.async {
            UIApplication.shared.registerForRemoteNotifications()
        }
    }
}

// Handle device token
func application(_ application: UIApplication,
                didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) {
    let token = deviceToken.map { String(format: "%02.2hhx", $0) }.joined()
    print("Device Token: \(token)")
}

// Local notification
let content = UNMutableNotificationContent()
content.title = "Reminder"
content.body = "Don't forget your task!"
content.sound = .default

let trigger = UNTimeIntervalNotificationTrigger(timeInterval: 60, repeats: false)
let request = UNNotificationRequest(identifier: UUID().uuidString, content: content, trigger: trigger)

UNUserNotificationCenter.current().add(request)

Core Data Example

import CoreData

// Define entity in .xcdatamodeld
// Person: name (String), age (Int16)

// Fetch data
func fetchPeople() -> [Person] {
    let context = persistentContainer.viewContext
    let fetchRequest: NSFetchRequest = Person.fetchRequest()
    
    do {
        let people = try context.fetch(fetchRequest)
        return people
    } catch {
        print("Fetch failed: \(error)")
        return []
    }
}

// Save data
func savePerson(name: String, age: Int16) {
    let context = persistentContainer.viewContext
    let person = Person(context: context)
    person.name = name
    person.age = age
    
    do {
        try context.save()
    } catch {
        print("Save failed: \(error)")
    }
}

// Delete data
func deletePerson(_ person: Person) {
    let context = persistentContainer.viewContext
    context.delete(person)
    
    do {
        try context.save()
    } catch {
        print("Delete failed: \(error)")
    }
}

Privacy & Security

  • App Tracking Transparency: Users must opt-in to tracking
  • Privacy Labels: App Store displays privacy practices
  • Data Encryption: End-to-end encryption for iMessage, FaceTime
  • Secure Enclave: Hardware-based key management
  • App Sandbox: Apps run in isolated environments
  • Code Signing: All apps must be signed by Apple
  • Privacy Report: Track app tracking and permissions
  • On-Device Processing: Siri, Photos analysis happen locally

App Store Submission

# Xcode steps
1. Archive your app (Product → Archive)
2. Validate the archive
3. Distribute to App Store Connect
4. Complete App Store metadata
5. Submit for review

# App Store Connect requirements
- App icon (1024x1024)
- Screenshots for all device sizes
- App description
- Keywords
- Privacy policy URL
- Support URL
- Age rating
- Pricing and availability

# Review process
- Typical review time: 24-48 hours
- Must comply with App Store Review Guidelines
- May require privacy justifications for permissions
